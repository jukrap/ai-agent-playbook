# Worklog Template

Choose the worklog language from the target system and user/team language. If the worklog will be pasted into a Korean Jira project, use the Korean section titles in the localized template. If the project or ticket is English-first, use the English section titles below.

## Title

`Short result-focused title`

## Summary
- Summarize the final result in one to three bullets.

## Background
- Explain why this work was needed.

## Observed Problem
- Describe the symptom, reproduction conditions, and blast radius.

## Cause
- Include only causes actually confirmed.

## Changes
- Focus on the result of the change. Do not only list file paths.

## Verification
- List commands, screens, and manual scenarios actually checked.

## Remaining Work
- Note remaining risks, blockers, or follow-up work.

## Writing Principles

- Do not simply list filenames, paths, or markdown document names.
- Explain the reason and result so another person can understand the work without separate materials.
- Prefer actual criteria and judgment over naming a person or external reference.
- Prioritize what was wrong and how it was organized over what was referenced.
