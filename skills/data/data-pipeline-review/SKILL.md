# Data Pipeline Review

Use this as the primary data skill for pipeline and analytics reliability.

## Workflow

1. Identify source systems, transformation steps, schedule, ownership, consumers, and freshness expectations.
2. Check schema drift, null handling, dedupe, late-arriving data, idempotency, backfill, and metric definitions.
3. Separate source-of-truth data from derived reports and generated artifacts.
4. Verify with row counts, reconciliation queries, sample records, and dashboard checks when possible.
