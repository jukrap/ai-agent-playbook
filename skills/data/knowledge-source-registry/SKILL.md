# Knowledge Source Registry

Use this as the primary data skill for registering project knowledge sources before search, ingestion, retrieval, or promotion work.

## Workflow

1. Identify source type, owner, use case, privacy tier, credentials boundary, update cadence, expected locator shape, and search/browse mode.
2. Record source status as available, building, partial, unavailable, failed, stale, or unknown.
3. Define locators and evidence envelopes before relying on search results, summaries, rows, objects, threads, or generated runtime reports.
4. Keep generated hits provisional until a reviewed fact, reference, map, or decision promotes them into durable memory.
