# Git Worklog Guardrails

Use this as the primary delivery skill for git, PR, and worklog safety.

## Workflow

1. Check branch, remote, upstream, dirty files, staged files, and local-only policy.
2. Stage only task-related files.
3. Verify with commands that the repository defines.
4. Write commit, PR, and worklog text from actual diff and actual verification.
5. Keep unrelated user changes untouched.

## Compatibility

`commit-worklog-guardrails` is a compatibility trigger for this skill.

## Reference

Read `references/git-delivery-checklist.md` before staging, committing, pushing, writing PR text, or recording a worklog.
