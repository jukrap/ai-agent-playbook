# Brand Identity System

Use this when design work needs a coherent identity system before UI implementation or marketing artifact production.

## Workflow

1. Inventory existing brand assets, product category expectations, audience, usage contexts, and required media.
2. Separate durable identity decisions from campaign-specific or screen-local styling.
3. Define typography, color, logo usage, iconography, imagery, tone, and application rules with accessibility and implementation constraints.
4. Map identity tokens into the repository's design system or handoff contract before adding one-off styles.
5. Verify that the identity can survive dark mode, responsive layouts, dense UI, generated images, social assets, and documentation artifacts.
