# Brand System Checklist

## Identity Inventory

- Brand name, wordmark, symbol, existing logo variants, favicon/app icon, and lockup constraints.
- Product category, audience, trust posture, and competitive visual norms.
- Required applications: web app, landing page, admin UI, mobile shell, social graphics, presentations, documents, email, charts, ads, and print.
- Existing design tokens, theme modes, component library, imagery, illustration, and icon sets.
- Legal, licensing, trademark, attribution, or client-specific restrictions.

## System Components

Define only what the current project can use and maintain:

- Logo usage: clear space, minimum size, background treatment, monochrome variant, and forbidden modifications.
- Typography: typefaces, fallback stacks, scale, weights, line heights, letter spacing, max line length, and responsive behavior.
- Color: primary, neutral, semantic, status, chart, focus, border, surface, overlay, and disabled states.
- Shape and spacing: radius, border, shadow, density, layout rhythm, and component spacing.
- Iconography: stroke weight, filled versus outline, sizing, alignment, and semantic meaning.
- Imagery: product screenshots, people, objects, generated images, photo treatment, crop rules, and alt-text expectations.
- Voice: terminology, tone, button language, error tone, empty states, and support messages.

## Application Matrix

For each application, record:

- required assets,
- size or aspect ratio,
- allowed token subset,
- dark/light behavior,
- accessibility constraints,
- export or implementation owner,
- verification evidence.

Examples of application rows: app header, primary button, chart palette, document cover, slide title, social avatar, app icon, product screenshot frame, and email banner.

## Review Gates

- The system can be implemented with existing tokens or a clear token migration.
- Identity colors meet contrast requirements in actual component states.
- Logo and icon assets have source, license, and export rules.
- Typography supports localization, long labels, dense tables, and mobile sizes.
- Decorative assets do not replace product evidence when users need to inspect the product.
- Generated or reference visuals are marked as evidence or drafts, not as durable source assets.
