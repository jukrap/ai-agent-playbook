# Product Type Decision Matrix

Use this reference when a design brief needs more than a vague style label. Start from the product category and user job, then choose layout pattern, information density, trust signals, visual tone, and approval checks.

## Decision Inputs

- Product category: SaaS, developer tool, dashboard, marketplace, portfolio, healthcare, finance, education, game, media, ecommerce, internal operations, or service booking.
- User job: compare, buy, learn, configure, monitor, create, approve, recover, explore, or decide.
- Primary pressure: trust, speed, clarity, delight, conversion, compliance, density, premium perception, or creative expression.
- Content shape: hero story, feature comparison, long-form editorial, grid catalog, data table, workflow builder, command palette, media gallery, or immersive canvas.
- Risk: regulated claims, accessibility, brand mismatch, performance budget, interaction complexity, sensitive data, or visual overstatement.

## Pattern Selection

| Product Type | Strong Default | Watch For |
| --- | --- | --- |
| B2B SaaS | task-focused product proof, clear CTA, compact feature evidence | oversized marketing hero that hides product reality |
| Developer tool | docs-first clarity, code/config proof, fast path to install | decorative dashboards with no runnable example |
| Internal/ops tool | dense scanning, tables, filters, status, audit trail | landing-page composition inside a work surface |
| Finance/security | trust-first hierarchy, restrained palette, evidence and control states | playful colors, vague claims, weak error/permission states |
| Healthcare/public service | accessibility, calm hierarchy, readable forms, confidence cues | low contrast, novelty motion, ambiguous actions |
| Marketplace/ecommerce | product/category visibility, comparison, price/availability clarity | atmospheric imagery that prevents inspection |
| Portfolio/editorial | strong art direction, readable story rhythm, selective motion | too many cards, tiny labels, or copied trend fragments |
| Game/interactive media | immediate playable/visual state, motion budget, responsive controls | static landing page before the primary experience |

## Style Dials

Set these before choosing colors or components:

- Visual density: sparse, balanced, dense, command-center.
- Motion intensity: none, subtle feedback, narrative transitions, continuous interactive scene.
- Geometry: sharp, small radius, pill, organic, technical.
- Surface model: flat, border-first, shadow-depth, glass, physical/tactile, full-bleed media.
- Typography posture: utilitarian, editorial, technical, luxury, playful, cinematic.
- Color role: neutral-first, one-accent, semantic, brand-led, media-led, category-coded.

## Anti-Pattern Checks

- A tool or dashboard uses a marketing-site hero before the actual work surface.
- A product page hides the product behind dark, blurred, or purely atmospheric media.
- A regulated or trust-heavy product uses playful effects that weaken credibility.
- A portfolio or editorial page uses generic SaaS cards instead of a distinct narrative rhythm.
- A 3D/canvas-heavy experience has no low-power, reduced-motion, or nonblank verification path.
- A design relies on copied brand tropes rather than project-owned tokens, content, and components.

## Pre-Delivery Design Check

- Layout proves the user job in the first viewport or first task surface.
- Type hierarchy is readable at mobile and laptop sizes.
- Interactive elements have hover, focus, disabled, loading, and error states when relevant.
- Color contrast and non-color cues support accessibility.
- Visual assets are inspectable and not only decorative.
- Responsive breakpoints preserve content priority instead of only shrinking the desktop layout.
- Claims, prices, permissions, and legal/compliance text are visible where users need them.
- Final handoff names which checks are verified, unknown, or intentionally deferred.
