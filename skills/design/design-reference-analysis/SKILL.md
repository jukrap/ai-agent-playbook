# Design Reference Analysis

Use this when reference material should inform design decisions without copying the reference into the product or public docs.

## Workflow

1. Capture the reference scope, source boundary, viewport, state, and why the reference is relevant.
2. Analyze hierarchy, layout, spacing, typography, color, components, motion cues, content density, and accessibility risks.
3. Separate reusable principles from distinctive source assets, copy, brand marks, and product-specific interaction patterns.
4. Convert useful findings into a local brief, design-system handoff, visual QA checklist, or implementation acceptance criteria.
5. Preserve evidence with portable locators or screenshots when allowed, but keep generated reports under runtime until reviewed.
