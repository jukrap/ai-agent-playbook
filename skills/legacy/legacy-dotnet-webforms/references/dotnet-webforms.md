# Dotnet WebForms Stack Checks

- Trace `.aspx`, code-behind, designer files, master pages, user controls, event handlers, and validation controls together.
- Check ViewState, postback order, data binding, Web.config, handlers/modules, IIS settings, and deployment transforms.
- Preserve control IDs, server-side event names, session keys, and framework version constraints.
- Verify initial load, postback, validation failure, auth/session, and deployed-framework compatibility when relevant.
