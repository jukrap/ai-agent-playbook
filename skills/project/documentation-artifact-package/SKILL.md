# Documentation Artifact Package

Package reviewed documentation into a bounded reader-facing deliverable with source evidence and maintenance rules.

## Workflow

1. Identify the package type: stakeholder brief, developer handoff, knowledge base entry, runbook bundle, report pack, or archive.
2. Define audience, purpose, owner, freshness date, update cadence, sensitivity boundary, and archive path.
3. Separate durable docs, generated runtime evidence, raw notes, and local-only sources before assembling the package.
4. Build a manifest of included artifacts, source evidence, caveats, decisions, open questions, and verification.
5. Keep generated reports under runtime until reviewed and explicitly promoted.
6. Add maintenance and stale-content rules so the package does not become hidden active policy.
