# Frontend UI Polish

Primary route: `frontend/ui-polish`.

Refine visible UI so it works well in the actual product context.

## Workflow

1. Inspect the existing screen, nearby components, design system, tokens, CSS conventions, and target audience.
2. Identify the workflow the user needs to complete, then improve hierarchy, spacing, layout stability, state coverage, and responsive behavior around that workflow.
3. Preserve the repository's style policy. Use `style-policy-selection` if the styling convention is unclear or contested.
4. Avoid decorative redesign unless the user asks for a new visual direction.
5. Verify changed surfaces with rendered UI or browser evidence when possible.
