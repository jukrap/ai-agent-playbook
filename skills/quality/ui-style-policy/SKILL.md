# UI Style Policy

Compatibility route: `frontend/style-policy-selection`.

Use this legacy trigger only for choosing or documenting the repository styling method. Do not use it for visual polish, responsive review, or broad design direction.

## Workflow

1. Route policy selection to `frontend/style-policy-selection`.
2. Inspect repository evidence before choosing design-system, CSS/class, utility-class, or inline-style policy.
3. Keep the choice narrow to the touched surface unless documenting a durable project rule.
4. Use `style-quality-review` or `ui-polish` only after the style policy is selected.

## Reference

Read `references/style-policy.md` when old project docs or prompts still point at `ui-style-policy`.
