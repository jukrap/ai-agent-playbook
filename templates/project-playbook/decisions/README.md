# Decisions

Use this folder for durable decisions that future work should not rediscover.

Suggested format:

```md
# YYYY-MM-DD Short Decision Title

## Decision
- What is now true.

## Context
- Why the decision was needed.

## Consequences
- What this enables.
- What this rules out or makes harder.

## Evidence
- Links to code, docs, worklogs, issues, or command output.
```

## Rules

- Record durable decisions, not every temporary preference.
- If a worklog contains a decision that remains current, promote it here.
- If a decision is reversed, add a new decision and link the old one.
