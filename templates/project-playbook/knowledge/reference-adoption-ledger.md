## Status Values

- `new`: seen but not reviewed.
- `reviewed`: analyzed but not yet adopted.
- `adopted`: converted into a local skill, workflow, MCP surface, validator, or reference.
- `deferred`: useful but not part of the current batch.
- `rejected`: not suitable for this harness or too noisy to carry forward.

## Rules

- Do not paste large upstream excerpts here.
- Do not include personal absolute paths, credentials, internal URLs, branch names, or PR numbers.
- Prefer capability names over upstream project names when the pattern has been generalized.
- Link only to local sanitized notes or adopted files.
