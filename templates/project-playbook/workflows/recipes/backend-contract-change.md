# Backend Contract Change

Inputs: API boundary, callers, payload examples, error behavior, compatibility needs.

Outputs: contract note, implementation plan, tests, migration or rollout notes.

Skills: API contract boundary, server-rendered change, security review when auth is involved.

Tools: `contracts check`, `contracts snapshot`, `operator search`, `operator map`.

Stop conditions: unknown caller behavior, missing schema owner, ambiguous error contract.

Verification: contract tests, integration tests, caller compatibility checks.
