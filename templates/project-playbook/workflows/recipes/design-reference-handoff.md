# Design Reference Handoff

Inputs: product surface, design intent, brand assets, reference screenshots or boards, Figma/mockup source, target screens, implementation boundary, accessibility target.

Outputs: design direction brief, reusable reference findings, brand/token notes, image-to-code or Figma handoff contract, visual evidence package, implementation acceptance criteria.

Skills: design brief direction, brand identity system, design reference analysis, image-to-code handoff, design system handoff, visual regression QA, frontend accessibility review.

Tools: reference capability matrix, reference adoption plan, browser or screenshot evidence, `qa_image_diff`, `operator preflight`, `index search`, `write_gate_preview`.

Stop conditions: unclear source authority, private or unlicensed visual material, missing product purpose, copied upstream branding risk, inaccessible color/motion direction, unsupported responsive target.

Verification: reference boundary recorded, visual evidence package captured when allowed, token/component mapping reviewed, desktop/mobile state matrix checked, accessibility and content-fit risks named before implementation.
