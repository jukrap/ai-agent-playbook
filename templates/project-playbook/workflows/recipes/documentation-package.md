# Documentation Package

Inputs: audience, artifact type, source evidence, changed behavior, user-facing scope, setup commands, owner, freshness date, translation needs, sensitive-data boundary, maintenance path, archive path, known constraints.

Outputs: PRD or lightweight spec, issue plan, release note or changelog, README or docs update, stakeholder package, developer handoff, knowledge package, map update, archive note, open-question list.

Skills: repo onboarding, project doc system, requirements PRD scope review, issue planning triage, release notes changelog, documentation artifact package, ADR spec handoff, context engineering memory design when durable memory is affected.

Tools: `workflow run-preview`, `playbook context`, `operator search`, `operator map`, `canon check`, `index status`, `write-gate preview`, `reference ledger check`, public-doc validation, translation validation, link checks, placeholder checks.

Stop conditions: unknown audience or artifact type, docs contradict code or current memory, unknown setup or verification path, unreviewed generated evidence, raw transcript promotion, unclear owner or maintenance path, private information risk, missing translation plan for translated docs.

Verification: source evidence review, public-doc hygiene, translation coverage, link/path check, placeholder check, setup command check, example validation against current files, maintenance/archive path review, canon promotion review when memory changes.
