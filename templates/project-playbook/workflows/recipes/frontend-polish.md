# Frontend Polish

Inputs: affected screen, user task, responsive targets, design constraints.

Outputs: UI changes, accessibility notes, visual QA evidence, verified checks.

Skills: UI polish, style policy, frontend accessibility review.

Tools: browser QA, screenshot diff, `operator preflight`, `write-gate preview`.

Stop conditions: missing target screen, unavailable assets, unclear design system boundary.

Verification: desktop and mobile review, overflow checks, loading/empty/error states, project tests.
