# Frontend Quality Review

Inputs: affected screens, user flows, supported breakpoints, state/data risks, accessibility targets.

Outputs: visual QA notes, accessibility findings, state/data risk notes, screenshots or issue list when useful.

Skills: UI polish, frontend state data flow, frontend accessibility review, visual regression QA.

Tools: browser QA, screenshot diff, `operator preflight`, `index search`, project test runner when available.

Stop conditions: missing runnable screen, unavailable test credentials, unclear design baseline, unstable data source without fixture.

Verification: desktop and mobile review, keyboard/focus checks, loading/empty/error states, stale data checks, overflow and visual diff checks.
