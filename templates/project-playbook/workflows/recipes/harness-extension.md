# Harness Extension

Inputs: capability gap, target category, proposed skill, reference, workflow, runtime command, MCP surface, adapter, permission tier, default-context impact, reference adoption source.

Outputs: taxonomy entry, skill/reference change, workflow recipe or runbook update, MCP resource/prompt/tool note, runtime artifact contract, docs, tests, translation update, install/sync expectation.

Skills: skill pack governance, context engineering memory design, runtime index cache design, agent skill authoring, MCP server design, reference adoption.

Tools: `capability catalog`, `skill catalog`, `workflow list`, `reference inventory`, `reference ledger check`, `index status`, `write-gate preview`, skill validation, translation validation, public-doc validation, sync dry run.

Stop conditions: duplicate skill name, unclear trigger, default prompt bloat, raw reference dump, unsafe write tier, missing schema or invalidation rule, unreviewed memory promotion, missing translation plan, docs/tests not updated with the change.

Verification: catalog check, workflow run-preview, MCP prompt/resource listing when relevant, syntax check, targeted tests, skill validation, translation validation, public-doc validation, install/update dry run, sync dry run.
