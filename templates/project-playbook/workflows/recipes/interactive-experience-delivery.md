# Interactive Experience Delivery

Inputs: rendered surface, renderer or media stack, asset sources, supported devices, interaction model, accessibility target, baseline or acceptance criteria.

Outputs: implementation plan, scene/rendering contract notes, visual evidence, interaction evidence, performance notes, and fallback/error-state notes.

Skills: interactive media 3D review, design system handoff, visual regression QA, frontend accessibility review, frontend state data flow when data-driven.

Tools: browser QA, screenshot or pixel check, console/network inspection, `workflow run-preview`, `write_gate_preview`, project test runner when available.

Stop conditions: blank or unmeasurable scene, unclear asset license/source, missing fallback for unsupported device/browser, unknown performance target, inaccessible pointer-only interaction.

Verification: nonblank screenshot or pixel check, desktop/mobile framing, primary interaction smoke test, asset load/error check, resize/device-pixel-ratio check, performance and cleanup review.
