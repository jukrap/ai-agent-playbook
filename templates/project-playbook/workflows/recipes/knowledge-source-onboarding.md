# Knowledge Source Onboarding

Inputs: source type, source owner, use case, privacy tier, credential boundary, update cadence, locator shape, search mode, browse mode, promotion policy.

Outputs: sources json entry, source status note, locator contract, evidence envelope example, browse instructions, freshness caveat, promotion rule, verification command.

Skills: knowledge source registry, knowledge retrieval pipeline review, context engineering memory design, runtime index cache design, data contract lineage review, pre-action fact gate.

Tools: `operator search`, `operator research`, `index status`, `index search`, `reference ledger check`, `canon check`, `diagnostics check`, source browse command when available.

Stop conditions: source owner is unknown, credential boundary is unclear, source range is unbounded, browse or reopen path is unavailable, freshness is unknown, locator cannot point to exact evidence, private payload would be committed, generated evidence would be trusted without review.

Verification: sources json entry validated, status and freshness recorded, sample locator reopened, scan range documented, credential value absent, privacy caveats recorded, promotion target reviewed.
