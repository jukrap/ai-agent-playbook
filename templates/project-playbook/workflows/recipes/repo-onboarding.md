# Repo Onboarding

Inputs: target repository, current task, available docs, known constraints.

Outputs: context notes, system map, verification command list, runtime index preview.

Skills: repo onboarding, project bootstrap, project doc system.

Tools: `catalog list`, `layout status`, `index build`, `operator map`, `operator audit`.

Stop conditions: missing repository, conflicting local instructions, unclear write scope.

Verification: run discovered project checks or document why they are unavailable.
