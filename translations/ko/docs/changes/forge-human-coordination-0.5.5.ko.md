# Forge 사람 중심 협업 구조 0.5.5

0.5.5는 기계 실행 그래프와 공개 협업 화면을 분리합니다. 세밀한 작업과 시도, 검증 근거, 재개 상태는 로컬 원장에 둡니다. GitHub 또는 Gitea에는 프로그램 로드맵 하나와 검토 가능한 전달 그룹만 보여줍니다.

## 공개 협업 구조

- `workflow.plan.v2`의 `coordination` 항목에 프로그램, 전달 그룹, 이슈·프로젝트 방식, 제목 정책, 하위 이슈 상한을 기록합니다.
- 원격 적용에는 검토된 협업 정보가 필요합니다. 기본 구조는 상위 이슈 하나와 최대 여섯 개 전달 그룹 이슈입니다. 작업별 이슈 방식은 이전 버전 호환 설정을 명시해야 쓸 수 있습니다.
- 관리 대상 한국어 제목은 계획에 명사형으로 작성해야 합니다. 문장형 종결은 기계적으로 바꾸지 않고 적용 전에 차단합니다.
- 상위 이슈와 그룹 이슈 본문에는 범위, 제외 범위, 수용 기준, 의존성, 검증, 위험, 복구, 진행률과 기술 상세를 담습니다.
- 관리 대상 영역만 바꾸고 사용자 메모는 보존합니다. GitHub 재조정은 확인된 상위·하위 이슈 관계만 사용하며 소유권을 확인할 수 없으면 중단합니다.
- 마일스톤에는 제목만 두지 않고 프로그램 목적, 완료 정의, 현재 작업·수용 기준 gate를 기록합니다.

## Projects, 라벨, 권한

GitHub Projects는 상태, 우선순위, 위험도, 단계, 진행률과 한국어 View를 담당합니다. 마일스톤은 릴리스 진척을 추적합니다. Projects를 쓰는 새 저장소에는 명시적 실행 승인용 `status:ready`만 만듭니다. 기존 `aapb:*` 라벨은 읽기 호환 별칭으로 유지합니다.

Managed Project에는 `AAPB Status`와 area field를 만듭니다. 상태 옵션은 Planned, Ready, In Progress, In Review, Blocked, Done입니다. Reconcile은 모든 Project item을 먼저 기록한 뒤 issue의 priority, risk, area label을 제거합니다. Project 또는 View 선행 작업이 실패하면 남은 원격 변경을 중단합니다.

Projects 쓰기가 필요한데 `project` 권한 범위가 없으면 doctor가 첫 원격 변경 전에 중단하고 다음 명령을 안내합니다.

```powershell
gh auth refresh -s project
aapb forge status .
```

인증 갱신은 자동으로 실행하지 않습니다. 이번 명령에서 Projects를 쓰지 않기로 했다면 `--allow-capability-fallback projects,views`로 마일스톤 대체 방식을 명시합니다.

## 재조정

`forge reconcile --plan`은 기존 작업 이슈를 전달 그룹으로 통합하는 변경을 미리 보여줍니다. 대체된 이슈를 닫고 하위 이슈 관계를 해제하려면 `--apply`와 `--allow-supersede`가 모두 필요합니다. 기록과 통합 대상 링크를 보존하며 이슈나 라벨을 삭제하지 않습니다.

`coordination.reconcile`에 지원 이슈와 기존 draft PR 번호를 지정하면 검토된 표시 구조 정리 대상으로 삼습니다. 하네스는 최신 CAS snapshot을 확인해 변경된 PR이나 이슈를 덮어쓰지 않습니다. GitHub Project preview는 roadmap, 전달 그룹, 지원 이슈, 검토 PR을 함께 보여줍니다. `project` scope가 없으면 모든 쓰기를 중단하고 예정 산출물 수와 브라우저 인증 해결 명령만 표시합니다.

초안 PR은 전달 그룹 제목을 사용하고 실제 변경, 검증 근거, 위험, 복구 방법과 남은 작업을 적습니다. 병합 승인 단계는 병합만 보류합니다. 구현, 검증, 초안 PR 생성 전에 먼저 병합하라는 조건을 만들지 않습니다.

변경 파일이 많거나 구조화 결과 또는 출력에서 확인한 테스트 수가 0이거나 UI 변경에 실제 미디어 증거가 없으면 검토 단계로 전환합니다. UI 증거는 해당 task 시도 중에 생성되고 controller 작업공간 안에 있어야 하며 형식 검증을 통과해야 합니다. PNG는 CRC와 압축 데이터를 확인합니다. 수용 기준에 `1600x1000`처럼 화면 크기를 적었다면 해당 크기의 PNG를 모두 확인합니다. 검증 명령에는 프로젝트 상대 경로인 `evidencePaths`를 지정할 수 있으며 검토 승인만으로 누락된 UI 증거를 건너뛸 수 없습니다.
