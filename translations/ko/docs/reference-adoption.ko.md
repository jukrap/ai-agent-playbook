# Reference Adoption

외부 참고 자료는 유용하지만, 일상적인 에이전트 지시에 긴 출처 목록이 그대로 남으면 노이즈가 됩니다. 참고 자료는 로컬 역량, 실행 절차, 간결한 참고 문서, 테스트, MCP 표면으로 정제해서 채택합니다.

목표는 참고 프로젝트의 이름이나 형태를 보존하는 것이 아닙니다. 목표는 에이전트 행동을 흐리는 개인 경로, 오래된 출처명, 중복 문장, 너무 넓은 지시를 들여오지 않고 이 하네스를 더 강하게 만드는 오래 쓸 운영 방식을 추출하는 것입니다.

## 채택 절차

1. 읽기 전용 레퍼런스 명령으로 원본 자료를 목록화합니다.
2. 각 항목을 역량 분류와 채택 상태로 나눕니다.
3. 무엇이든 채택하기 전에 대표 파일을 직접 읽습니다.
4. 오래 쓸 실무 절차, 제약, 판단 기준, 실패 양상, 검증 항목을 추출합니다.
5. 짧은 트리거 지침은 `SKILL.md`에 둡니다.
6. 재사용 절차는 해당 스킬의 `references/`에 둡니다.
7. 반복 가능한 절차는 워크플로우 레시피로 추가합니다.
8. 런타임 탐색이나 실행이 필요한 동작만 MCP 리소스, 프롬프트, 도구로 추가합니다.
9. 채택한 규칙이 구조적이라면 테스트나 검증기를 추가합니다.
10. 주요 설계 결정은 문서나 작업 로그에 기록합니다.

## 채택 단계

| 단계 | 사용할 때 | 산출물 |
|---|---|---|
| Observe | 넓게 훑고 역량 매트릭스를 검토할 때. | 계획 노트, 채택 장부, 스킬 변경 없음. |
| Distill | 참고 자료에 유용한 절차는 있지만 런타임 기능은 필요 없을 때. | 스킬 참고 문서, 워크플로우 레시피, 문서 갱신. |
| Operationalize | 참고 자료가 반복 가능한 점검이나 생성 증거를 설명할 때. | CLI/MCP 미리보기, 검증기, 런타임 산출물 스키마, 테스트. |
| Govern | 참고 자료가 하네스 성장 방식 자체를 바꿀 때. | 분류 규칙, 린트, 권한 모델, 유지보수 문서, 마이그레이션 메모. |

## 추출할 것

- 트리거 조건: 에이전트가 해당 역량을 선택해야 하는 상황.
- 전제 조건: 필요한 저장소 신호, 도구, 권한, 자격 정보, 사용자 승인.
- 중단 조건: 멈추거나 질문하거나 상향 판단하거나 거절해야 하는 상황.
- 증거 계약: 스크린샷, 명령 출력, 위치 정보, 추적 정보, 보고서, 장부, 검증 매트릭스.
- 경계 규칙: 내용이 스킬, 참고 문서, 템플릿, 런타임, 메모리, 문서, MCP 중 어디에 속하는지.
- 실패 양상: 빈 캔버스, 오래된 메모리, 숨은 프롬프트 주입, 중복 전달, 위험한 설정, 의존성 드리프트, 문서와 코드 불일치.

## 추출하지 않을 것

- 최종 내용이 해당 프로젝트 자체를 다루는 경우가 아니라면 원본 프로젝트의 브랜드나 이름.
- 재사용 운영 규칙이 아니라 사람용 글에 가까운 긴 문장.
- 특정 공급자를 다루는 스킬이 아니라면 공급자 전용 설정.
- 명시적인 권한 모델 없이 대상 작업 공간 밖에 쓰는 명령.
- 개인 절대 경로, 내부 도메인, 자격 정보, 브랜치명, 이슈 번호, PR 번호, 오래된 운영 상태.

## 제외 기준

- 개인 절대 경로.
- 회사명과 내부 URL.
- 자격 정보, 토큰, 브랜치명, 이슈 번호, PR 번호.
- 트리거 시점에 유용하지 않을 만큼 긴 원문 복사 자료.

## Ledger 원칙

- 레퍼런스 장부는 원본 상태를 추적하는 용도로만 쓰고, 일상적인 프롬프트 맥락으로 쓰지 않습니다.
- 오래 쓸 내용이 로컬 스킬, 문서, 워크플로우, 테스트, MCP 표면에 반영된 뒤에만 채택됨으로 표시합니다.
- 너무 좁거나, 노이즈가 많거나, 오래되었거나, 라이선스가 불분명하거나, 이미 다룬 참고 자료는 건너뜀으로 표시합니다.
- 출처 표기나 라이선스 준수에 필요한 경우가 아니라면 공개 스킬 본문에 원본별 메모를 남기지 않습니다.
- 생성된 런타임 보고서는 검토된 승격 단계 전까지 `memory/`와 분리합니다.

## 검토 체크리스트

- 이 추가가 에이전트가 올바른 행동을 더 빨리 선택하게 합니까?
- 내용이 스택명 또는 출처명 중심이 아니라 역량 중심입니까?
- 재사용 세부사항이 `SKILL.md`를 부풀리지 않고 `references/`에 있습니까?
- 한국어 번역은 반쪽짜리 영어 자리표시자가 아니라 자연스러운 한국어입니까?
- 새 MCP 쓰기 동작은 명시적 선택, 미리보기 우선, 경로 검증, 감사 기록을 갖췄습니까?
- 원문과 번역을 고친 뒤 검증을 실행했습니까?
