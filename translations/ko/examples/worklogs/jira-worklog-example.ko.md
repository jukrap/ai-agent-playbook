# Jira Worklog 예시

한국어 Jira ticket, sprint note, team-facing work record에 올릴 때는 이 형태를 사용합니다. 영어 원문은 범용 Jira 예시이며, 이 번역본은 한국어 Jira에 맞게 section title과 문장 톤을 조정한 버전입니다.

### 제목

`배송조회 필터와 상세 모달 상호작용 정리`

### 요약

- 조회 조건 입력 방식과 상세 진입 방식을 사용자 흐름 기준으로 다시 맞췄다.
- 모바일과 데스크톱에서 같은 기능이 다른 모양으로 보여도 동작은 일관되도록 정리했다.

### 배경

- 조회 조건이 여러 입력칸으로 흩어져 있어 공간을 많이 차지했고, 작은 화면에서는 버튼과 입력 요소가 충돌했다.
- 카드 전체를 눌러 상세가 열리는 방식은 사용자가 기대한 동작과 달라, 운송장 번호 링크 중심의 진입 방식으로 바꿀 필요가 있었다.

### 확인된 문제

- 작은 화면에서 조회 필터가 줄바꿈되며 우측 버튼이 일부 가려졌다.
- 카드 클릭 범위가 넓어 상세 진입 의도가 없는 터치에서도 모달이 열릴 수 있었다.
- 모달을 열었을 때 배경 스크롤과 좌우 여백이 불안정하게 보였다.

### 원인

- 입력 요소와 액션 버튼의 우선순위가 정해지지 않아 화면 폭이 줄어들면 같은 줄에서 공간을 과도하게 차지하고 있었다.
- 카드 전체에 클릭 affordance가 남아 있어, 실제 상세 진입 포인트가 무엇인지 시각적으로도 불명확했다.
- 모달 정렬 기준과 스크롤 잠금 방식이 단일 기준으로 묶여 있지 않았다.

### 적용 내용

- 조회 조건은 날짜 범위, 검색 기준 선택, 검색어 입력, 상태 선택, 검색 실행 순서로 다시 구성했다.
- 상세는 카드 전체가 아니라 운송장 번호 링크를 눌렀을 때만 열리도록 바꿨다.
- 모달은 중앙 정렬, 배경 스크롤 잠금, 모바일 여백 기준을 하나의 방식으로 통일했다.

### 검증

- `pnpm test:run`
- `pnpm lint`
- `pnpm build`

### 잔여 확인 사항

- 실기기에서 모바일 키보드와 날짜 선택 레이어가 겹치지 않는지 추가 확인이 필요하다.
- 태블릿 중간 폭에서 카드 줄바꿈과 배지 위치를 한 번 더 미세조정할 수 있다.
