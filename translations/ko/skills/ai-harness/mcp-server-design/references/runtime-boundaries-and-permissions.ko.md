# MCP 런타임 경계와 권한

MCP 서버가 작은 표준 입출력 래퍼를 넘어 장기 실행 데몬, HTTP/SSE 전송, 큐 처리, 원격 터널, 승인 흐름, 색인 기반 검색 연결, 자격 증명을 다루는 런타임을 포함할 때 이 참고 자료를 사용합니다.

## 런타임 형태

- 프로토콜 파싱, 전송/세션 관리, 도구 실행, 응답 형식을 분리합니다. 외부에 보이는 조정 계층이 이를 묶을 수는 있지만, 각 계층은 따로 테스트할 수 있어야 합니다.
- 세션 상태를 명확히 둡니다: 세션 ID, 전송 방식, 등록된 도구, 대기 중인 호출, 대기 중인 응답, 정리 동작.
- 큐 처리는 모든 도구 안에 숨어 있는 분기가 아니라 별도 실행 전략으로 다룹니다. 요청이 접수, 대기, 완료, 실패, 폐기 중 어디에 있는지 기록합니다.
- 닫힌 전송, 취소된 요청, 유휴 데몬, 오래된 대기 응답, 버전 불일치 재시작을 정리하는 경로를 둡니다.

## 로컬 데몬 경계

- 로컬 보조 서버는 기본적으로 루프백 주소에만 바인딩합니다.
- 포트, 프로세스 ID, 토큰, 소켓 경로가 들어 있는 로컬 데몬 상태 파일은 소유자만 읽을 수 있게 저장합니다.
- 상태 확인은 루트 토큰, 비밀값, 로컬 경로를 유출하지 않고 생존 여부만 증명해야 합니다.
- 버전 또는 바이너리 불일치는 오래된 코드가 새 명령을 처리하게 두지 말고 깨끗하게 재시작합니다.
- 로그는 크기를 제한하고 자격 증명 값, 쿠키 값, 비공개 프롬프트 원문, 마스킹되지 않은 도구 입력을 포함하지 않아야 합니다.

## 원격 또는 터널 표면

- 터널 또는 포트 전달을 통해 로컬 데몬 전체 표면을 노출하지 않습니다.
- 원격 접근에는 별도 리스너, 경로 허용 목록, 또는 이에 준하는 단단한 경계를 둡니다.
- 루트/관리자 토큰은 원격 터널 표면에서 동작하면 안 됩니다. 원격 호출자가 사용할 수 있는 좁은 명령 묶음에는 범위 제한 토큰을 발급합니다.
- 알 수 없는 경로는 명확히 거부하고, 거부 이유를 크기가 제한된 감사 로그에 기록합니다.
- 클라이언트가 일반 인증 헤더를 보낼 수 없는 이벤트 스트림에는 별도 제한 세션 방식이 필요합니다.

## 승인과 쓰기 게이트

- 승인 도구는 초기화, 기능 목록 조회, 요청, 판단, 알 수 없는 메서드, 잘못된 입력, 거부 상태 테스트를 갖춘 프로토콜로 모델링합니다.
- 기본 거부와 자동 거부 모드는 실제 외부 서비스 없이 테스트할 수 있어야 합니다.
- 쓰기 가능한 MCP 도구는 서버 측 활성화, 검증된 대상 경로, 사전 실행 미리보기, 명시적 `apply`, 지속되는 감사 근거를 요구해야 합니다.
- 도구 설명은 서버가 읽기 전용 모드로 실행 중일 때 쓰기 권한을 암시하면 안 됩니다.

## 검색/읽기 연결

- 검색용 MCP 도구는 최소화합니다. 후보를 찾는 도구 하나와 정확한 근거를 읽는 도구 하나로 충분한 경우가 많습니다.
- 허용된 출처 범위는 클라이언트가 보낸 질의 매개변수에만 맡기지 말고 서버에서 강제합니다.
- 검색 결과에는 다시 열 수 있는 위치 정보, 범위, 최신성 메모, 주의점을 함께 반환합니다.
- 색인, 캐시, 임베딩은 파생 산출물로 다룹니다. 출처 목록이나 장기 기억 문서가 무엇이 권위 있는 원본인지 말해야 합니다.

## 자격 증명과 설정 경계

- `env:`와 `file:` 자격 증명 참조는 얇은 클라이언트나 에이전트 프롬프트가 아니라 신뢰된 서버 실행 환경에서 해석합니다.
- 메타데이터에는 원문 값 대신 자격 증명 참조와 권한 범위를 저장합니다.
- 어느 환경이 설정, 큐 상태, 캐시, 색인, 커넥터 자격 증명을 소유하는지 문서화합니다.
- 누락, 만료, 권한 부족, 폐기, 형식 오류가 있는 자격 증명 사례를 검증합니다.

## 검증

- 프로토콜 테스트: 초기화, 도구/리소스 목록 조회, 도구 호출, 알 수 없는 메서드, 잘못된 매개변수, 오류 응답 형식.
- 권한 테스트: 읽기 전용 모드가 쓰기를 숨기거나 거부하고, 활성화 모드에서도 `apply`를 요구하는지 확인합니다.
- 세션 테스트: 대기 응답 정리, 취소된 요청, 닫힌 전송, 큐 작업자 실패, 재연결.
- 원격 표면 테스트: 루트 토큰 거부, 알 수 없는 경로 거부, 허용된 범위 제한 명령, 감사 로그 기록.
- 검색/읽기 테스트: 허용 범위, 거부 범위, 누락된 출처, 정확한 범위 읽기, 오래된 색인 경고, 쓰기 없음.
