# 잡과 워커 신뢰성

비동기 백엔드 작업을 일반 요청 코드가 아니라 전달 계약으로 다룹니다.

## 작업 흐름

1. 트리거, 큐/토픽/스케줄, 워커 프로세스, 동시성 모델, 임대/가시성 제한 시간, 종료 동작을 확인합니다.
2. 전달 방식을 예약 실행, 큐, 이벤트 기반, 웹훅 후속 작업, 일괄 처리, 재실행, 수동 복구 중 하나로 분류합니다.
3. 멱등성 키, 지속 상태 전이, 재시도 가능한 오류, 영구 실패, 데드레터 경로, 재실행 동작을 정의합니다.
4. 트랜잭션, 비교 후 갱신, 고유 제약, 체크포인트, 제공자 작업 ID로 부수 효과를 보호합니다.
5. 중복 전달, 재시도, 영구 실패, 부분 일괄 실패, 종료, 재실행, 관측성을 검증합니다.
6. 공유 서비스/모듈 변경은 `backend-change-safety`, 제공자 콜백은 `connector-integration-change`와 함께 사용합니다.

## 참고 자료

워커 루프, 큐 소비자, 스케줄러, 웹훅 후속 잡, 오래 실행되는 일괄 작업을 편집하기 전에 `references/job-worker-reliability.md`를 읽습니다.
