# 잡과 워커 신뢰성

백엔드 잡, 워커, 큐, 소비자, 예약 작업, 재시도, 데드레터 처리, 재실행, 오래 실행되는 작업에 이 참고 자료를 사용합니다.

## 런타임 목록화

- 트리거: 크론, 스케줄러, 큐, 토픽, 스트림, 웹훅, 명령, 백필, 마이그레이션, 수동 복구.
- 워커 런타임: 단일 프로세스, 풀, 서버리스 트리거, 컨테이너 복제본, 사이드카, 앱 서버 스레드, 외부 워크플로 엔진.
- 전달 모델: 최대 한 번, 최소 한 번, 정확히 한 번이라는 주장, 순서가 있는 파티션, 순서 없는 일괄 처리, 지연 재시도, 수동 재실행.
- 조정 수단: 잠금, 임대, 가시성 제한 시간, 체크포인트, 커서, 고유 제약, 자문 잠금, 외부 제공자 ID.
- 부수 효과: 데이터베이스 쓰기, 외부 API 호출, 이메일, 과금, 파일/객체 쓰기, 캐시 무효화, 이벤트, 알림, 감사 기록.

## 신뢰성 규칙

- 상태 기계를 명시합니다: 대기, 임대됨, 처리 중, 성공, 재시도 가능 실패, 영구 실패, 데드레터, 취소, 보상, 이미 적용됨.
- 되돌릴 수 없는 부수 효과 전에 멱등성 키를 저장합니다. 제공자 이벤트 ID, 업무 자연 키, 작업 ID, 정규화 페이로드 해시를 선호합니다.
- 로컬 아키텍처가 허용한다면 메시지 파싱, 검증, 상태 전이, 부수 효과, 최종 확인을 분리합니다.
- 큐 계약에 따라 지속 캡처 또는 성공적인 부수 효과 뒤에만 메시지를 확인하거나 삭제합니다.
- 오래 실행되는 스캔, 가져오기, 백필, 내보내기에는 체크포인트를 사용합니다.
- 독성 메시지 처리를 명시합니다. 일괄 계약이 그렇다고 말하지 않는 한 나쁜 항목 하나가 전체 묶음을 조용히 버리면 안 됩니다.
- 재실행은 안전하고 관측 가능하며 ID 범위, 시간 범위, 테넌트, 작업 ID 같은 필터로 제한되도록 설계합니다.

## 종료와 동시성

- 정상 종료가 처리 중인 작업을 마저 처리하거나 다시 큐에 넣는지 확인합니다.
- 임대 연장과 가시성 제한 시간이 최악의 처리 시간보다 길거나 갱신 가능한지 확인합니다.
- 여러 워커가 같은 작업을 중복 적용할 수 없는지 확인합니다.
- 워커 전용 설정이 HTTP 전용 요청/세션 도우미 안에 숨지 않았는지 확인합니다.
- 재시도 폭주 때 자원 제한, 호출 제한, 역압 동작을 확인합니다.

## 관측성

- 큐 길이, 큐 대기 시간, 재시도 수, 데드레터 수, 워커 지연, 처리 시간, 성공/실패 수, 멈춘 상태를 추적합니다.
- 로그와 추적에는 메시지 ID, 작업 ID, 시도 횟수, 최종 상태, 정리된 실패 코드를 포함합니다.
- 알림은 적체 증가, 독성 메시지, 제공자 장애, 코드 회귀, 데이터별 복구를 구분해야 합니다.

## 검증

- 중복 전달 테스트: 같은 메시지는 지속 결과 하나와 명확한 중복 신호 하나를 만듭니다.
- 재시도 테스트: 일시 실패는 되돌릴 수 없는 부수 효과를 중복하지 않고 재시도됩니다.
- 영구 실패 테스트: 잘못된 페이로드나 거부된 권한은 예상한 실패 또는 데드레터 상태로 갑니다.
- 부분 일괄 테스트: 실패한 항목 하나가 다른 항목을 망가뜨리거나 잃거나 숨기지 않습니다.
- 종료 테스트: 부수 효과 전후의 중단이 재개되거나 명확한 복구 경로를 보고합니다.
- 재실행 테스트: 제한된 범위의 재실행은 멱등적이며 검토 가능한 근거를 남깁니다.
