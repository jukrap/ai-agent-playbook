# 요청 검증과 오류 계약

백엔드 검증이나 오류 처리를 바꾸기 전에 요청과 실패 응답 계약을 보호합니다.

## 작업 흐름

1. 라우트, 처리기, 스키마, DTO, 미들웨어, 예외 매퍼, 클라이언트 호출자, 문서화된 오류 형태를 확인합니다.
2. 파싱, 검증, 권한, 업무 규칙 실패, 충돌, 호출 제한, 예상하지 못한 오류 경로를 구분합니다.
3. 명시적으로 버전이 나뉜 변경이 아니라면 상태 코드, 오류 코드, 필드 경로, 다국어 키, 재시도 의미를 안정적으로 유지합니다.
4. 오류 본문, 로그, 추적, 검증 메시지에 민감한 값을 넣지 않습니다.
5. 성공, 잘못된 입력, 권한 없음, 충돌, 찾을 수 없음, 예상하지 못한 실패 경로를 검증합니다.
6. 프론트엔드 목, 생성된 클라이언트, OpenAPI, 외부 소비자가 형태에 의존한다면 `api-contract-boundary`와 함께 사용합니다.

## 참고 자료

검증기, DTO, 예외 필터, 미들웨어, 오류 응답 스키마를 편집하기 전에 `references/request-error-contract.md`를 읽습니다.
