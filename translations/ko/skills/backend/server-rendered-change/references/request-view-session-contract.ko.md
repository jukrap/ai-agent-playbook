# 요청/뷰/세션 계약

동작이 서버와 뷰 계층에 나뉘어 있는 백엔드 렌더링 페이지, 폼, 컨트롤러, 템플릿, 리다이렉트 흐름을 바꿀 때 이 참고 자료를 사용합니다.

## 전체 흐름 추적

수정하기 전에 요청을 그립니다.

- 라우트, HTTP 메서드, 경로 변수, 질의 매개변수, 폼 필드, 파일, 헤더, 쿠키, CSRF 토큰.
- 미들웨어, 필터, 인터셉터, 가드, 세션 로더, 언어/테마 처리, 인가.
- 컨트롤러/액션, 서비스 호출, 저장 호출, 매퍼 SQL, 저장 프로시저, 외부 연동 호출.
- 모델/뷰 키, 템플릿 조각, 부분 템플릿, include, 레이아웃 슬롯, 클라이언트 강화 훅, 검증 메시지 위치.
- 리다이렉트 대상, 플래시 메시지, 세션 변경, 캐시 헤더, 오류 페이지.

## 계약 보존

- 작업이 명시적으로 바꾸는 경우가 아니라면 요청 매개변수 이름, 필드 이름, 바인딩 이름, 모델/뷰 키, 세션 키, 리다이렉트 경로를 보존합니다.
- 여러 라우트가 포함하는 템플릿 조각과 부분 템플릿은 공유 계약으로 다룹니다.
- 렌더링 흐름이 의존하는 매퍼 SQL, 저장 프로시저, 보고서 질의도 페이지 계약의 일부로 다룹니다.
- 검증 실패 동작을 안정적으로 유지합니다: 제출 값, 오류 메시지, 포커스 대상, 상태 코드, 세션/플래시 상태.
- 이전 클라이언트나 북마크 URL이 있으면 경로를 제거할 때 리다이렉트 또는 호환 라우트를 보존합니다.

## 상태와 인가

- 비로그인, 권한 없음, 만료된 세션, 역할 불일치, 오래된 CSRF, 여러 탭 동시 사용 사례를 확인합니다.
- 흐름이 세션 상태, 플래시 메시지, 자동 로그인 쿠키, 기능 플래그, 서버 캐시를 읽거나 쓰는지 검증합니다.
- 인가 판단을 템플릿이나 클라이언트 스크립트로 옮기지 않습니다.
- 숨은 필드가 가격, 역할, 소유자, 상태, 권한의 신뢰 기준이 되지 않게 합니다.

## 렌더링과 상호작용

- 빈 상태, 긴 콘텐츠, 번역된 문구, 검증 오류, 성공, 권한 거부 상태를 확인합니다.
- 클라이언트 스크립트가 페이지를 강화한다면 프로젝트가 기대하는 경우 스크립트 없음 또는 서버 전용 fallback을 검증합니다.
- 점진적 강화 훅을 안정적으로 유지합니다: 요소 이름, ID, 데이터 속성, 엔드포인트 가정.
- 다운로드, 인쇄, 보고서, 내보내기는 콘텐츠 타입, 파일명, 인코딩, 페이지 나눔, 브라우저 동작을 검증합니다.

## 검증

- GET 렌더 성공.
- POST 성공과 리다이렉트 대상.
- 사용자 입력이 보존되는 POST 검증 실패.
- 인증/세션 만료와 권한 없는 접근.
- 공유되는 경우 템플릿 조각 재사용 경로.
- 데이터 형태가 바뀌면 저장 프로시저, 매퍼 SQL, 보고서 출력.
- 라우트가 이동하면 이전 라우트, 북마크, 호환 리다이렉트.

## 중단 조건

- 폼 필드, 모델 키, 세션 키, 리다이렉트 대상이 바뀌는데 호출자와 템플릿을 추적하지 않았습니다.
- 검증 실패 동작을 검증하지 않았습니다.
- 인가가 템플릿 표시 여부로만 확인됩니다.
- 흐름이 저장 프로시저나 매퍼 SQL에 의존하는데 검토하지 않았습니다.
- 배포 중 이전/새 코드가 호환되지 않는 뷰/세션/데이터 계약을 함께 사용해야 합니다.
