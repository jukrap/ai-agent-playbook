# 파이프라인 정합성과 보강

분석, ETL, 배치, 보고, 데이터 제품 파이프라인이 기준 데이터에서 조용히 어긋날 수 있을 때 이 참고 자료를 사용합니다.

## 파이프라인 목록화

변경하기 전에 경로를 그립니다.

- 출처: 데이터베이스 테이블, 이벤트, 파일, API, 스트림, 수동 업로드, 외부 내보내기.
- 변환: 추출, 정규화, 조인, 보강, 중복 제거, 집계, 모델 점수화, 게시.
- 런타임: 일정, 트리거, 재시도 정책, 동시성, 파티션, 워터마크, 체크포인트, 소유자.
- 산출물: 테이블, 마트, 대시보드, 보고서, 내보내기, 알림, 기계 소비자, 수동 업무 흐름.
- 계약: 입도, 기본 키, 시간 창, 최신성 목표, 보존 기간, 필드 의미, 지표 정의.

## 계약과 변화 점검

- 출처 소유자, 기준 입도, 기본 식별자, 시간대, 통화, 분모, null 의미를 확인합니다.
- 추가, 삭제, 이름 변경, 타입 변경, 늦게 도착한 데이터, 중복, 삭제, 정정 레코드를 확인합니다.
- 원천 필드와 파생 필드, 생성된 근거를 분리합니다.
- 문서화되지 않은 필드, 암묵적 필터, 적재 순서, 이전 이름에 의존하는 소비자를 식별합니다.
- 스키마 변화는 단순 매퍼 불편이 아니라 계약 사건으로 추적합니다.

## 멱등성과 재실행

- 작업은 같은 파티션이나 체크포인트에 반복 실행되어도 중복된 업무 효과가 없어야 합니다.
- 보강 작업에는 경계가 있는 파티션 범위, 정렬, 예상 행 수, 재개 표시, 검증 질의가 필요합니다.
- 재실행은 과거 행을 교체, 병합, 추가, 정정 중 어떻게 처리하는지 명시해야 합니다.
- 부분 실패는 영향받은 파티션, 실패 레코드, 재시도 횟수, 하위 최신성 영향을 노출해야 합니다.
- 파이프라인이 대시보드나 마트에 쓰면 데이터 복구와 시각/보고 캐시 새로고침을 분리합니다.

## 품질 신호

- 파티션별 행 수, 출처 대비 산출물 정합성, 중복 키 검사, null/범위 검사, 참조 무결성 검사, 최신성 지연.
- 지표 점검: 분모, 분자, 제외 조건, 시간 창, 시간대, 통화, 세그먼트.
- 이상치, 갑작스러운 하락, 급증, 불가능한 값에 대한 분포 점검.
- 알림 경로: 소유자, 심각도, 기대 대응, 묵음 시간, 알려진 오탐 조건.

## 대시보드와 보고서 정합성

- 파이프라인 변경 뒤 대표 대시보드/보고서를 적어도 하나 검증합니다.
- 출처 합계, 변환 합계, 대시보드 합계, 사용자가 의존하는 내보내기 합계를 비교합니다.
- 보이는 숫자가 출처 정정, 로직 변경, 늦은 데이터, 필터 변경, 캐시 새로고침 중 무엇 때문에 바뀌었는지 기록합니다.
- 스크린샷, 질의 결과, 생성 요약은 검토되기 전까지 런타임 근거로 둡니다.

## 중단 조건

- 기준 데이터, 소유자, 입도, 최신성 목표를 모릅니다.
- 보강 또는 재실행이 행을 중복하거나 정정 데이터를 조용히 덮어쓸 수 있습니다.
- 지표 정의가 바뀌는데 소비자 검토가 없습니다.
- 파이프라인 성공이 대시보드/보고서 최신성을 증명하지 못합니다.
- 원천 비공개 데이터, 자격 증명, 내부 연결 정보가 재사용 문서에 들어가야 합니다.
