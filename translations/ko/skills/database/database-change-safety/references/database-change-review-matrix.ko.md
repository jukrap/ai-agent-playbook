# 데이터베이스 변경 검토 매트릭스

데이터베이스 변경이 단일 안전 추가 마이그레이션보다 넓거나 영향 범위가 불분명할 때 이 참고 자료를 사용합니다.

## 범위 지도

수정하기 전에 영향을 받는 표면을 확인합니다.

- 테이블, 컬럼, 색인, 제약, 뷰, 트리거, 저장 프로시저, 함수, 시퀀스, 시드 데이터, 예약 작업.
- 애플리케이션 읽기/쓰기 경로, 배치 작업, 보고서, 내보내기, 대시보드, 검색 색인, 캐시, 외부 소비자.
- 데이터 소유자, 운영 소유자, 롤백 소유자, 승인 경로, 예상 행 수, 피크 읽기/쓰기 트래픽.
- 배포 형태: 단일 배포, 롤링 배포, 블루/그린, 여러 서비스 버전 공존, 모바일/클라이언트 버전 공존, 수동 DBA 실행.

## 변경 분류

- 추가형: null 허용 컬럼, 선택 테이블, 비차단 색인, 새 뷰, 선택 시드 행.
- 호환성 민감형: 이름 변경, 타입 변경, enum/domain 변경, not null 기본값, unique 제약, 외래 키, 생성 컬럼, 루틴 시그니처, 트리거 동작.
- 파괴형: drop, delete, truncate, 되돌릴 수 없는 재작성, 손실 변환, 마이그레이션 후 정리.
- 보강형: 기존 행 업데이트, 정합성 복구, 중복 제거, 파생 필드 재계산, 외부 출처 복구.
- 보고서 민감형: 숫자를 조용히 바꿀 수 있는 질의, 뷰, 저장 프로시저, 내보내기, 마트, 대시보드, 지표 변경.

## 확장/축소 순서

- 먼저 확장합니다. 기존 읽기/쓰기 경로가 계속 동작하는 동안 새 구조를 추가합니다.
- 이중 읽기 또는 이중 쓰기는 일관성, 재실행, 롤백 전략이 분명할 때만 사용합니다.
- 보강 작업에는 정렬 키, 배치 크기, 재개 지점, 재시도 동작, 진행 확인 질의를 둡니다.
- 중간 스키마가 이전/새 버전을 모두 지원한 뒤 애플리케이션 읽기와 쓰기를 이동합니다.
- 축소는 나중에 합니다. 소비자가 이동했고 정리가 명시적으로 승인된 뒤 이전 컬럼, 뷰, 루틴, 호환 코드만 제거합니다.

## 온라인 안전성 점검

- 잠금 동작, 색인 생성 방식, 테이블 재작성 위험, 복제 지연, 트랜잭션 시간, 마이그레이션 제한 시간을 추정합니다.
- 기본값, 제약, 트리거, 생성 컬럼이 기존 행을 재작성하거나 쓰기 경로를 막는지 확인합니다.
- 대량 업데이트는 배치, 트랜잭션 크기, 대기/속도 조절, 재시도 표시로 경계를 둡니다.
- 저장 프로시저와 매퍼 SQL은 시그니처, 매개변수 순서, 결과 형태, 오류 동작, 호출자 호환성을 검증합니다.
- 시드 또는 기준 데이터는 비공개 값을 넣지 않고 멱등성과 환경별 값을 검증합니다.

## 데이터 정합성과 보고

- 전후 행 수, null 수, 중복 검사, 고아 데이터 검사, 정합성 질의를 정의합니다.
- 마이그레이션 성공뿐 아니라 대표 보고서, 내보내기, 대시보드, 하위 작업을 검증합니다.
- 지표 정의 변경, 분모 변경, 시간대/통화 가정, 알려진 제외 조건을 기록합니다.
- 생성된 질의 결과는 사람이 검토해 장기 문서로 승격하기 전까지 런타임 근거로 둡니다.

## 롤백 근거

- 롤백이 DDL 되돌리기, 백업 복원, 보상 마이그레이션 실행, 기능 플래그 비활성화, 작업 중단, 읽기 경로 되돌리기 중 무엇인지 명시합니다.
- 파괴형 변경은 실행 전 마지막 안전 지점과 백업/내보내기 근거를 이름 붙입니다.
- 보강 작업은 영향을 받은 배치를 멈추고, 재개하고, 복구하고, 되돌리는 방법을 기록합니다.
- 호환성 민감형 변경은 배포 중 버전 공존이 가능하면 이전 앱과 새 스키마, 새 앱과 이전 스키마를 검증합니다.

## 중단 조건

- 행 규모, 잠금 동작, 의존 소비자를 모릅니다.
- 파괴 작업에 명시적 승인과 롤백 메모가 없습니다.
- 보강 작업이 경계 없거나, 멱등적이지 않거나, 진행 표시가 없습니다.
- 보고서 출력이 바뀌는데 소비자 검토가 없습니다.
- 사용 중인 배포 형태에서 이전/새 애플리케이션 버전이 중간 스키마를 모두 견딜 수 없습니다.
