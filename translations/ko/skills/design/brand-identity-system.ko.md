# Brand Identity System

UI implementation 또는 marketing artifact production 전에 coherent identity system이 필요할 때 씁니다.

## Workflow

1. Existing brand asset, product category expectation, audience, usage context, required media를 조사합니다.
2. Durable identity decision을 campaign-specific 또는 screen-local styling과 분리합니다.
3. Typography, color, logo usage, iconography, imagery, tone, application rule을 accessibility와 implementation constraint와 함께 정의합니다.
4. One-off style을 추가하기 전에 identity token을 repository design system 또는 handoff contract로 매핑합니다.
5. Identity가 dark mode, responsive layout, dense UI, generated image, social asset, documentation artifact에서도 유지되는지 검증합니다.
