# Design Brief Direction

시각 구현 전에 product intent, audience, style direction, acceptance target이 명확하지 않을 때 쓰는 primary design skill입니다.

## Workflow

1. Product type, user job, audience expectation, brand posture, existing design asset, constraint, implementation surface를 확인합니다.
2. 개별 color, font, effect, component를 고르기 전에 coherent direction을 먼저 정합니다.
3. Visual variance, motion intensity, information density style dial을 정해 이후 구현의 공유 목표를 만듭니다.
4. 이 제품을 generic, off-brand, inaccessible, hard-to-maintain하게 만들 anti-pattern을 명명합니다.
5. Brand identity, reference analysis, design-system handoff, frontend polish, visual QA로 라우팅 가능한 brief를 만듭니다.
