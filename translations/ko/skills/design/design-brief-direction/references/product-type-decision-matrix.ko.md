# 제품 유형 결정 매트릭스

디자인 요약서에 막연한 스타일 이름보다 더 구체적인 판단이 필요할 때 이 참고 자료를 사용합니다. 제품 유형과 사용자가 해야 할 일에서 시작해 배치 패턴, 정보 밀도, 신뢰 신호, 시각 톤, 승인 점검을 정합니다.

## 판단 입력

- 제품 유형: SaaS, 개발자 도구, 대시보드, 마켓플레이스, 포트폴리오, 의료, 금융, 교육, 게임, 미디어, 전자상거래, 내부 운영, 서비스 예약.
- 사용자 과업: 비교, 구매, 학습, 설정, 모니터링, 생성, 승인, 복구, 탐색, 결정.
- 핵심 압력: 신뢰, 속도, 명확성, 즐거움, 전환, 규정 준수, 밀도, 고급감, 창의적 표현.
- 콘텐츠 형태: 첫 화면 이야기, 기능 비교, 긴 글 편집, 격자형 카탈로그, 데이터 표, 워크플로우 빌더, 명령 팔레트, 미디어 갤러리, 몰입형 캔버스.
- 위험: 규제 대상 주장, 접근성, 브랜드 불일치, 성능 예산, 상호작용 복잡도, 민감 데이터, 시각적 과장.

## 패턴 선택

| 제품 유형 | 강한 기본값 | 주의할 점 |
| --- | --- | --- |
| B2B SaaS | 작업 중심 제품 증거, 명확한 행동 버튼, 압축된 기능 근거 | 실제 제품이 보이지 않는 과대형 마케팅 첫 화면 |
| 개발자 도구 | 문서 우선의 명확성, 코드/설정 근거, 빠른 설치 경로 | 실행 가능한 예시가 없는 장식용 대시보드 |
| 내부/운영 도구 | 밀도 높은 훑기, 표, 필터, 상태, 감사 기록 | 업무 화면 안의 랜딩 페이지식 구성 |
| 금융/보안 | 신뢰 우선 계층, 절제된 팔레트, 근거와 제어 상태 | 장난스러운 색, 모호한 주장, 약한 오류/권한 상태 |
| 의료/공공 서비스 | 접근성, 차분한 계층, 읽기 쉬운 양식, 확신을 주는 신호 | 낮은 대비, 새로움만 노린 움직임, 모호한 동작 |
| 마켓플레이스/전자상거래 | 제품/카테고리 가시성, 비교, 가격/재고 명확성 | 검토를 방해하는 분위기용 이미지 |
| 포트폴리오/편집물 | 강한 아트 방향, 읽기 좋은 이야기 리듬, 절제된 움직임 | 너무 많은 카드, 작은 라벨, 복사한 유행 조각 |
| 게임/상호작용 미디어 | 즉시 보이는 플레이/시각 상태, 움직임 예산, 반응형 조작 | 핵심 경험 전에 나오는 정적 랜딩 페이지 |

## 스타일 기준

색상이나 컴포넌트를 고르기 전에 다음을 정합니다.

- 시각 밀도: 여백 중심, 균형형, 고밀도, 관제실형.
- 움직임 강도: 없음, 미세한 피드백, 이야기 전환, 지속 상호작용 장면.
- 형태: 날카로움, 작은 둥근 모서리, 알약형, 유기적, 기술적.
- 표면 모델: 평면, 테두리 중심, 그림자 깊이, 유리 느낌, 물리적/촉각적, 전체 폭 미디어.
- 타이포그래피 태도: 실용적, 편집적, 기술적, 고급, 장난스러움, 영화적.
- 색상 역할: 중립색 우선, 한 가지 강조색, 의미 색상, 브랜드 주도, 미디어 주도, 유형별 색상.

## 위험 패턴 점검

- 도구 또는 대시보드가 실제 작업 화면보다 마케팅 사이트 첫 화면을 먼저 보여줍니다.
- 제품 페이지가 어둡고 흐리거나 분위기만 있는 미디어 뒤에 제품을 숨깁니다.
- 규제 대상 또는 신뢰 중심 제품이 신뢰도를 약화시키는 장난스러운 효과를 사용합니다.
- 포트폴리오 또는 편집 페이지가 고유한 이야기 리듬 대신 일반적인 SaaS 카드만 사용합니다.
- 3D/캔버스 중심 경험에 저전력, 움직임 감소, 빈 화면 방지 검증 경로가 없습니다.
- 디자인이 프로젝트 고유 토큰, 콘텐츠, 컴포넌트 대신 복사한 브랜드 관습에 의존합니다.

## 인계 전 디자인 점검

- 첫 화면 또는 첫 작업 표면에서 사용자 과업이 드러납니다.
- 글자 계층이 모바일과 노트북 크기에서 읽힙니다.
- 필요한 상호작용 요소에 hover, focus, disabled, loading, error 상태가 있습니다.
- 색상 대비와 색상 외 단서가 접근성을 지원합니다.
- 시각 자산은 검토 가능하며 장식에만 머무르지 않습니다.
- 반응형 분기점이 데스크톱 배치를 줄이기만 하지 않고 콘텐츠 우선순위를 보존합니다.
- 주장, 가격, 권한, 법무/준수 문구가 사용자가 필요한 위치에 보입니다.
- 최종 인계가 검증 완료, 미확인, 의도적 보류 항목을 구분합니다.
