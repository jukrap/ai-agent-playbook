# Three.js WebGL 운영 점검

렌더링 화면이 Three.js, React Three Fiber, 원시 WebGL, 셰이더 배경, 파티클 시스템, 모델 뷰어, WebGL 터미널, 캔버스 중심 인터랙티브 장면을 사용할 때 이 참고 자료를 사용합니다.

## 장면 계약

- 구현 전에 화면의 약속을 정의합니다: 주 피사체, 카메라 대상, 시작 프레임, 컨트롤, 로딩 상태, 빈 상태, 미지원 브라우저 상태, 모션 감소 상태.
- 기본 3D 장면은 모호하지 않아야 합니다. 사용자는 숨은 구현 설명을 읽지 않아도 무엇을 보여 주는지 알아야 합니다.
- 하나의 마운트된 화면에는 하나의 장면 소유자를 둡니다. 공유 렌더러 풀은 크기 변경, 해제, 컨텍스트 손실, 라우트 전환 생명주기 관리가 이미 있을 때만 사용합니다.
- 사용자가 검사, 복사, 탭 이동, 번역을 해야 하는 컨트롤, 라벨, 범례, 설명, 핵심 값은 불투명한 캔버스 밖에도 둡니다.
- 장식용 WebGL 배경은 향상 요소로 취급합니다. 본문 가독성, 탐색, 배터리 절약 모드, 정적 스크린샷 캡처를 방해하면 안 됩니다.

## 렌더러와 카메라

- `aspect-ratio`, `min-height`, `max-height`, 전체 화면 규칙처럼 명확한 CSS 제약으로 캔버스 경계를 고정합니다. 로딩 문구나 컨트롤, 동적 라벨이 장면 컨테이너 크기를 흔들면 안 됩니다.
- 프로젝트에 더 강한 성능 예산이 없다면 렌더러 픽셀 비율은 보통 `Math.min(window.devicePixelRatio, 2)`처럼 상한을 둡니다.
- 색상 관리와 톤 매핑은 사용하는 Three.js 버전에 맞춰 확인합니다. 코드가 `outputColorSpace`, 구식 `outputEncoding`, `ACESFilmicToneMapping`, 커스텀 후처리 중 무엇을 쓰는지 확인합니다.
- 카메라의 near/far plane, 대상, 시야각, 모델 스케일은 눈대중이 아니라 실제 자산 경계에서 정합니다.
- 크기가 바뀔 때 렌더러 크기, 카메라 비율과 투영 행렬, composer 크기, 화면 공간 uniform을 함께 갱신합니다.

## 자산과 머티리얼

- 모델, 텍스처, HDRI, 셰이더, 폰트, 비디오의 출처와 라이선스, 크기를 기록합니다. 불투명한 원격 자산은 캐시, 대체 상태, CORS 동작 없이 추가하지 않습니다.
- 리뷰 전에 GLB/GLTF 지오메트리, 텍스처 크기, 텍스처 형식, 애니메이션 클립, 환경 맵 크기를 압축하고 제한합니다.
- 반복 mesh에는 객체별 변경이 필요한 경우가 아니라면 geometry와 material 인스턴스를 공유합니다.
- 시각 목표를 만족하는 가장 단순한 material을 우선합니다. 조명이 필요 없는 UI 마커에는 `MeshBasicMaterial`도 적절하며, 물리 기반 material은 조명, 환경, 톤을 함께 확인해야 합니다.
- 모델, 텍스처, 셰이더, 미디어 로딩 실패에는 보이는 대체 상태가 있어야 합니다. 빈 캔버스는 허용 가능한 대체 상태가 아닙니다.

## 애니메이션과 상호작용

- 애니메이션 루프는 가시성과 생명주기에 묶습니다. 언마운트, 숨겨진 탭, 모달 닫힘, 비활성 라우트, 저전력 모드에서는 requestAnimationFrame을 멈추거나 낮춥니다.
- `prefers-reduced-motion`을 존중하고, 세션 중 설정이 바뀌면 즉시 반영합니다.
- 장면이 단순 장식이 아니라 작업 흐름의 일부라면 포인터 컨트롤에는 키보드나 DOM 대체 경로가 있어야 합니다.
- 카메라 컨트롤에는 최소/최대 거리, polar angle, pan 제한, damping, reset, 대화상자나 메뉴 활성 중 비활성 상태를 둡니다.
- 게임이나 시뮬레이션은 프로젝트가 커스텀 엔진을 명시적으로 요구하지 않는 한 검증된 물리 또는 규칙 라이브러리를 사용합니다.

## 성능 예산

- 초기 자산 바이트, 첫 비어 있지 않은 프레임까지의 시간, 안정 프레임률, draw call, 삼각형 수, 텍스처 메모리, 상호작용 중 CPU/GPU 급등을 추적합니다.
- 일반 제품 UI에서 무제한 파티클, shadow, 후처리, 반사 material, 고분할 geometry를 피합니다.
- 품질 단계는 기기 성능이나 사용자 선호와 연결될 때만 추가합니다. 또 다른 시각 프리셋보다 저전력 또는 정적 경로가 더 유용한 경우가 많습니다.
- 반복 마운트와 언마운트, 라우트 전환 후 메모리를 확인합니다. geometry, material, texture, render target, control, event listener 누수가 흔한 운영 장애입니다.
- 장면이 대시보드, 에디터, WebView 안에서 실행된다면 단독 전체 화면 데모가 아니라 실제 주변 UI 부하 아래에서 테스트합니다.

## 검증

- 캔버스가 비어 있지 않고 올바르게 프레이밍되며 컨트롤이나 문구를 가리지 않는다는 데스크톱과 모바일 스크린샷을 남깁니다.
- 기본 장면이 빈 출력이면 실패하는 픽셀 검사나 스크린샷 차이를 둡니다.
- 셰이더 컴파일 오류, 누락 자산, CORS 실패, WebGL 경고, 컨텍스트 손실을 브라우저 콘솔과 네트워크에서 확인합니다.
- hover/select, drag/orbit, zoom, reset, 키보드 경로, 터치 제스처, 모션 감소 모드 같은 기본 컨트롤을 실제로 조작합니다.
- 화면을 벗어났다가 돌아오거나 컴포넌트를 다시 마운트하거나 표면을 닫았다 여는 방식으로 콘솔 잡음과 메모리 증가를 보며 정리를 검증합니다.
