# 스타일 정책 선택

눈에 보이는 UI를 바꾸기 전에 저장소의 스타일링 방식을 근거로 선택합니다.

## 작업 흐름

1. 프로젝트 문서, 공유 UI 기본 요소, 토큰, 기존 컴포넌트, 스타일시트, 유틸리티 클래스, 인라인 스타일 사용을 확인합니다.
2. 공유 컴포넌트, 변형, 슬롯, 토큰이 화면을 책임한다면 기존 디자인 시스템을 먼저 사용합니다.
3. 저장소가 이미 쓰는 방식에 맞춰 CSS/클래스, 유틸리티 클래스, 인라인 스타일 객체 중 하나를 고릅니다.
4. 한 작업을 위해 병렬 스타일링 방식을 새로 만들지 않습니다.
5. 프로젝트에 명확한 지침이 없고 나중에도 중요할 때만 루트 지침이나 `.ai-agent-playbook/`에 오래 남을 정책을 기록합니다.
6. 정책을 고른 뒤 렌더링 품질 점검에는 `style-quality-review`, `ui-polish`, `visual-regression-qa`를 사용합니다.

## 참고 자료

스타일 관례가 애매하거나 충돌하거나 문서화할 가치가 있을 때 `references/style-policy-selection.md`를 읽습니다.
