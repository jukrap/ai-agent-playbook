# UI Polish

보이는 UI 품질을 위한 기본 프론트엔드 스킬입니다.

## 작업 절차

1. 제품 흐름, 기존 디자인 시스템, 주변 컴포넌트, 대상 화면 너비를 확인합니다.
2. 실제 작업 주변의 계층, 간격, 레이아웃 안정성, 텍스트 맞춤, 접근성, 상태 범위를 개선합니다.
3. 사용자가 새 시각 방향을 요청하지 않았다면 무관한 재설계는 피합니다.
4. 가능하면 관련 데스크톱/모바일 너비에서 렌더링된 UI를 검증합니다.

## 참고 자료

폴리시 작업에 작업 적합성, 계층, 간격, 넘침, 접근성 상태, 반응형 동작, 시각 검증을 구체적으로 점검해야 하면 `references/task-surface-polish-check.md`를 읽습니다.

## 호환성

`frontend-ui-polish`는 이 스킬의 호환 트리거입니다.
