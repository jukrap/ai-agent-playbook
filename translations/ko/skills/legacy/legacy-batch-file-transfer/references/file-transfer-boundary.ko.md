# 파일 전송 경계

레거시 배치 작업, 예약 가져오기/내보내기, 파일 드롭, 파트너 전송에 사용합니다.

## 계약 필드

- 생산자와 소비자.
- 전송 방식: 로컬 폴더, 공유 드라이브, SFTP, 이메일 첨부, object storage 또는 수동 업로드.
- 파일명 패턴, 확장자, 타임스탬프, 순번, 충돌 동작.
- 인코딩, 구분자, 따옴표 처리, 줄 끝, 헤더 row, trailer row.
- 필수 컬럼, 선택 컬럼, 기본값, 컬럼 순서.
- 시간대, 영업일, 휴일, 마감 시각 규칙.

## 실행 수명 주기

1. 후보 파일을 감지합니다.
2. 파일을 잠그거나 점유합니다.
3. 부수 효과 전에 형식을 검증합니다.
4. 명확한 성공/실패 집계와 함께 record를 처리합니다.
5. 파일을 문서화된 상태로 보관, 격리 또는 유지합니다.
6. 부분 실패를 식별할 수 있는 로그와 경고를 냅니다.
7. 안전한 재실행 또는 대사를 지원합니다.

## 실패 모드

- 다른 프로세스가 아직 쓰는 중인 파일을 부분적으로 읽습니다.
- 재시도나 운영자 업로드로 중복 파일이 들어옵니다.
- 인코딩 불일치가 식별자나 이름을 바꿉니다.
- 헤더 변경이 위치 기반 파싱을 밀어냅니다.
- 시간대나 휴일 규칙이 잘못된 영업일을 선택합니다.
- 일부 record가 실패했는데도 성공 상태가 남습니다.

## 검증 세트

- 정상 파일.
- 빈 파일.
- 잘못된 파일.
- 중복 파일.
- 예상 운영 물량에 가까운 대용량 파일.
- 성공 후 재실행.
- 부분 실패 후 재실행.

## 복구

보관/복구 경로가 문서화되지 않았다면 파일을 덮어쓰거나 삭제하지 않습니다. 복구가 수동이면 완료를 증명하는 운영자 단계와 필요한 근거를 정확히 기록합니다.
