# 레거시 시스템의 기능 단위

주변 아키텍처를 바꾸지 않고 레거시 시스템에 동작을 추가할 때 사용합니다.

## 단위 형태

기존 흐름을 기준으로 기능을 정의합니다.

- 사용자 진입점 또는 예약 트리거.
- 입력 필드, 검증, 오류 표시.
- 서버 action, 서비스 함수, 저장 프로시저 또는 파일 처리기.
- 저장, 내보내기, 출력, 큐 또는 downstream 산출물.
- 권한, 세션, 감사, 운영 복구.

## 계약 확장

- 필드는 명시적으로 추가하고 모든 바인딩 지점을 추적합니다.
- 의도적인 계약 버전이 아니라면 기존 필드 이름과 형식을 보존합니다.
- 기존 row, 파일, 소비자와 호환되도록 nullable/default 동작을 유지합니다.
- 같은 데이터에 의존하는 보고서, 내보내기, 가져오기, 배치 작업을 갱신합니다.
- 구현되지 않은 백엔드 또는 제품 결정은 막힌 지점으로 문서화합니다.

## 구현 가드레일

- 같은 종류의 기능에 가장 가까운 기존 패턴을 재사용합니다.
- 공유 동작이 증명될 때까지 새 코드는 새 동작 범위에 묶습니다.
- 기능 하나 때문에 새 상태 모델, 프레임워크, 서비스 경계를 도입하지 않습니다.
- 누락된 서버 동작을 가리는 목 데이터를 추가하지 않습니다.
- 바뀐 경계 가까이에 호환성 점검을 둡니다.

## 검증

- 인접한 기존 경로가 계속 작동합니다.
- 새 정상 경로가 작동합니다.
- 검증 실패 또는 업무 규칙 실패가 사용자나 운영자에게 보입니다.
- 인증/세션/권한 동작이 유지되거나 의도적으로 바뀝니다.
- 건드렸다면 저장, 내보내기, 출력, 큐 또는 downstream 산출물을 확인합니다.

## Handoff

작지 않은 변경에는 다음을 기록합니다.

- 사용한 기존 패턴.
- 확장한 계약.
- 확인한 소비자.
- 검증 근거.
- 남은 위험과 되돌림 메모.
