# 보고서와 출력 계약

보고서, 인쇄 출력, 내보내기, 라벨, 청구서, 장치별 출력을 바꾸기 전에 사용합니다.

## 계약 표면

- 데이터 쿼리와 필터 매개변수.
- 템플릿 원본과 서식 계층.
- 출력 필드, 컬럼 순서, 그룹, 합계, 정렬.
- 용지 크기, 여백, 방향, DPI, 글꼴, 바코드/QR 요구사항.
- locale, timezone, 숫자 서식, 날짜 서식, 통화 정밀도.
- 내보내기 라이브러리, 브라우저 인쇄 경로, 프린터 드라이버, 장치 경로, 용지/라벨 stock.

## 소비자 점검

- 사람 독자.
- 재무, 감사, 창고, 지원 또는 다른 운영 팀.
- downstream spreadsheet macro 또는 upload.
- 바코드 스캐너, 라벨 프린터, 청구서 프린터 또는 제3자 수신자.
- 보관, 법적, 컴플라이언스 유지 절차.

## 검증 사례

- 대표 정상 출력.
- 긴 텍스트와 줄바꿈.
- 빈 값과 null 값.
- 많은 row와 여러 페이지 출력.
- 경계 합계와 반올림.
- 바코드/QR 스캔 가독성.
- 경로가 둘 이상이면 Excel/PDF/인쇄 출력.

## 실패 모드

- 화면 미리보기와 실제 인쇄 출력이 다릅니다.
- 글꼴 대체가 페이지 나눔을 바꿉니다.
- 컬럼 순서 변경이 downstream spreadsheet 또는 import를 바꿉니다.
- locale 또는 timezone이 보고되는 영업일을 바꿉니다.
- 프린터, macro, 보관 문서 프로세스가 사용하는 템플릿 필드가 제거됩니다.

검증에 사용한 정확한 브라우저, 내보내기 형식, 프린터 경로, 샘플 데이터, 환경을 기록합니다.
