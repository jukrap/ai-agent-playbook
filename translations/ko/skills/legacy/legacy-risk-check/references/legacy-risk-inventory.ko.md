# 레거시 위험 목록

공유 레거시 동작을 바꾸기 전이나 위험한 레거시 변경 완료를 선언하기 전에 사용합니다.

## 목록 점검

- 진입: 경로, 명령, 예약 작업, 수동 운영, 브라우저 화면 또는 파트너 파일.
- 상태: 세션, 쿠키, 숨은 입력, 전역 변수, local storage, 데이터베이스 row, 임시 파일 또는 캐시.
- 계약: 요청 필드, 응답 필드, 내보내기 컬럼, 파일 레이아웃, 프로시저 매개변수 또는 템플릿 변수.
- 공유 자산: CSS, JavaScript, include, 레이아웃 템플릿, 유틸리티, 태그, 생성 산출물.
- 실행 선택: feature flag, 서버 설정, 브라우저 모드, 배포 폴더, classpath, include path 또는 환경 switch.
- 운영: 되돌림, 백업, 수동 보정, 큐 재처리, 배치 재실행, 지원 스크립트, 릴리스 패키징.

## 고위험 패턴

- 공유 선택자나 클래스가 관련 없는 화면을 제어합니다.
- 폼 필드가 이름으로 서버에 바인딩됩니다.
- 저장 프로시저 결과 형태가 보고서, 내보내기, 애플리케이션 코드에서 함께 소비됩니다.
- 생성 파일이 커밋되거나 배포 중 수동 복사됩니다.
- 일정 작업이나 수동 지원 절차가 실패 상태를 복구하는 유일한 방법입니다.
- 브라우저나 장치 연동을 현재 환경에서 테스트할 수 없습니다.

## 위험 결정

변경을 분류합니다.

- 낮음: 활성 경로 하나, 명확한 소유자 하나, 로컬 검증 가능, 되돌림이 단순합니다.
- 중간: 공유 자산이나 계약을 건드리지만 소비자를 알고 검증할 수 있습니다.
- 높음: 활성 실행 경로가 불명확하거나 소비자를 모르거나 데이터 마이그레이션이 있거나 검증에 없는 인프라가 필요합니다.

## 중간/높음 위험 handoff

- 무엇을 바꿨는지.
- 어떤 소비자를 확인했는지.
- 무엇을 검증하지 못했는지.
- 어떤 명령, 시나리오, 샘플 파일, 쿼리, 환경을 사용했는지.
- 어떻게 되돌리거나 격리할 수 있는지.
