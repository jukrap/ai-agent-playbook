# 서버 렌더링 레거시 흐름

템플릿, 컨트롤러, 폼, 세션, 혼합 클라이언트 동작이 있는 서버 렌더링 페이지에 사용합니다.

## 흐름 지도

- 경로와 HTTP method.
- 컨트롤러/action/filter/middleware 경로.
- 뷰 모델, 템플릿 변수, 레이아웃, 부분 템플릿, include.
- 폼 action, 필드 이름, 숨은 입력, CSRF token, 검증 오류.
- redirect, flash message, 세션 변경, 권한 확인.
- JavaScript 보강, AJAX 분기, fallback 동작.

## 마크업과 스타일 점검

- 마크업 변경 전 공유 레이아웃과 부분 템플릿.
- 클래스나 ID 변경 전 전역 CSS 선택자.
- 정리 전 브라우저별 스타일, 출력 스타일, 오래된 CSS hack.
- 스크립트 로드 순서, 인라인 handler, 위임 이벤트, 플러그인 초기화.
- 클라이언트 검증 변경 전 서버 렌더링 오류 상태.

## 폼 계약 점검

- 필드 이름과 서버 바인딩.
- 다중 값 필드와 checkbox 기본 동작.
- 흐름 상태에 쓰이는 숨은 입력.
- 날짜, 숫자, locale parsing.
- 파일 업로드 enctype, 크기 제한, 서버 저장 경로.
- 성공 후 redirect와 검증 실패 후 재표시.

## 검증 시나리오

- GET 최초 로드.
- 성공 POST.
- 검증 실패 POST.
- 인증/세션 만료 또는 권한 실패.
- 제출 후 브라우저 뒤로가기/다시 로드.
- 프로젝트가 지원하면 JavaScript 비활성 또는 degraded path.

보이는 페이지 변경만 조용히 처리하지 말고 백엔드/템플릿 불확실성을 기록합니다.
