# ADR Spec Handoff

Durable decision, spec, handoff documentation을 위한 primary project skill입니다.

## Workflow

1. 무엇이 바뀌었고, 왜 중요하며, 누가 의존하고, durable project memory가 어디에 있어야 하는지 확인합니다.
2. Current rule, product fact, decision, open question, worklog history, generated evidence를 분리합니다.
3. Reviewed stable fact만 ADR, spec, map, runbook, `.ai-playbook` memory로 승격합니다.
4. Raw chat transcript, noisy reference, personal path, credential, branch name, PR number는 durable doc에서 제외합니다.

## Reference

ADR scope, decision record, alternative, consequence, status에는 `references/adr-decision-capture.md`를 읽습니다.

Worklog, plan, handoff, evidence를 durable project memory로 승격할 때는 `references/spec-worklog-promotion.md`를 읽습니다.
