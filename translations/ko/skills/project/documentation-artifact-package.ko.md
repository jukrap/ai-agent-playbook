# Documentation Artifact Package

검토된 documentation을 source evidence와 maintenance rule이 있는 bounded reader-facing deliverable로 package합니다.

## 작업 흐름

1. Package type을 확인합니다. stakeholder brief, developer handoff, knowledge base entry, runbook bundle, report pack, archive 중 무엇인지 구분합니다.
2. Audience, purpose, owner, freshness date, update cadence, sensitivity boundary, archive path를 정의합니다.
3. Package를 만들기 전에 durable docs, generated runtime evidence, raw notes, local-only source를 분리합니다.
4. Included artifact, source evidence, caveat, decision, open question, verification manifest를 만듭니다.
5. Generated report는 검토와 명시적 승격 전까지 runtime 아래에 둡니다.
6. Package가 숨은 active policy가 되지 않도록 maintenance와 stale-content rule을 추가합니다.

## Reference

Package contents, evidence, caveat, boundary를 정의할 때는 `references/artifact-package-manifest.md`를 읽습니다.

Package를 reader에 맞추고 update 또는 archive responsibility를 정할 때는 `references/reader-handoff-and-maintenance.md`를 읽습니다.
