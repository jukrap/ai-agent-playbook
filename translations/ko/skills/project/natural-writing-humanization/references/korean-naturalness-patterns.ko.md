# 한국어 자연스러움 점검

README, 문서, 배포 노트, PR 본문, 안내문, 번역본을 한국어답게 다듬을 때 사용합니다.

## 먼저 볼 것

- 제품명, 명령어, 파일 경로, 오류 메시지, 숫자, 날짜, 버전, 위험 경고는 원문 그대로 보존합니다.
- 영어 문장 순서를 그대로 옮기지 말고, 한국어에서 자연스러운 주어와 행동을 앞에 둡니다.
- 영어 용어는 명령어, API 이름, 고유명사, 업계에서 더 익숙한 용어일 때만 남깁니다.
- 처음 등장하는 영어 용어만 간단히 설명하고, 이후에는 짧고 일관된 표현을 씁니다.
- 안내문이나 운영 문서는 긴 문장 하나보다 짧은 사실 문장 여러 개가 더 낫습니다.

## 번역투 신호

- `할 수 있습니다`, `것으로 보입니다`, `제공됩니다`, `진행됩니다`, `수행됩니다`가 반복됩니다.
- `이를 통해`, `뿐만 아니라`, `또한`, `따라서`, `관점에서`, `측면에서`가 문단마다 나옵니다.
- `중요한 역할을 합니다`, `기여합니다`, `최적의 경험을 제공합니다`, `강력한 기능을 제공합니다`처럼 근거 없는 추상 표현이 많습니다.
- 한국어 문장 안에 영어 명사 묶음이 과하게 들어갑니다.
- 바로 지시해도 되는 운영 문서에서 지나치게 돌려 말합니다.

## 고치는 방법

- `이를 통해 사용자는 X할 수 있습니다`는 필요하면 `사용자는 X합니다`로 줄입니다.
- `A뿐만 아니라 B도 제공합니다`는 두 문장으로 나누거나 더 중요한 주장만 남깁니다.
- 수동형이 반복되면 주체를 드러냅니다. 예: `빌드가 실행됩니다`보다 `CI가 빌드를 실행합니다`.
- `강력한 검증` 같은 표현은 실제 동작으로 바꿉니다. 예: `경로 이탈, 바이너리 파일, 큰 파일을 거부합니다`.
- 한국어 요약에는 약간의 말맛을 남겨도 됩니다. 모든 문장을 같은 길이와 구조로 맞추지 않습니다.

## 검토 질문

- 이 글은 시스템을 이해한 사람이 한국어로 쓴 것처럼 읽히는가?
- 독자가 영어 표현을 해석하느라 다음 행동을 놓치지 않는가?
- 문서 전체에서 용어가 일관되는가?
- 수정하면서 경고, 명령어, 수치, 한계를 약하게 만들지는 않았는가?
