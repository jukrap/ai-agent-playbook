# 말투 보존과 경계

자연스러운 글 다듬기는 편집입니다. 저자성을 세탁하거나 근거를 바꾸는 작업이 아닙니다.

## 보존할 것

- 의미, 사실, 숫자, 날짜, 버전 범위, 명령어, 파일 경로, API 필드, 법적 의미, 보안 경고, 배포 범위.
- 작성자의 유용한 말투: 단호함, 조심스러움, 격식 수준, 도메인 용어, 의도적인 반복.
- 독자와 목적: 설치 안내, 장애 기록, PR 본문, 배포 노트, 이해관계자 브리프, 튜토리얼, 공개 README.

## 개선할 것

- 흔한 AI식 표현과 번역투를 줄입니다.
- 반복되는 문장 구조와 과하게 대칭적인 bullet을 줄입니다.
- 과장된 형용사를 실제 동작으로 바꿉니다.
- 긴 문장을 더 명확하게 나누되 정확성을 잃지 않습니다.
- 한국어 번역본은 영어 문법을 옮긴 문장이 아니라 한국어 문서처럼 읽히게 합니다.

## 하지 말 것

- 저자성, 진정성, 인간 작성 여부를 주장하지 않습니다.
- AI 탐지기, 시험, 채용 절차, 운영 정책, 검수 기준을 우회하도록 돕지 않습니다.
- 개인 경험, 감정, 자격, 출처, 인용을 지어내지 않습니다.
- 경고, 법적 고지, 보안 위험, 제한, 되돌림 조건을 약하게 만들지 않습니다.
- 운영 문서를 마케팅 문구처럼 과하게 다듬지 않습니다.

## 멈추거나 물어볼 때

- 원문 사실이 모호하거나 숫자가 맞지 않거나 독자가 불명확합니다.
- 법적, 보안, 의료, 금융, 계약, 컴플라이언스 성격의 주장을 크게 다시 써야 합니다.
- 요청한 톤이 프로젝트 정책, 브랜드 말투, 독자의 필요와 충돌합니다.
- 원문이 이미 충분히 간결하고 자연스럽습니다. 이때는 큰 수정이 필요 없다고 말합니다.
