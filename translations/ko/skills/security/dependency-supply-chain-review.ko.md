# Dependency Supply Chain Review

의존성, SBOM, 라이선스, 출처 증명, 패키지 공급망 위험을 검토할 때 쓰는 기본 보안 스킬입니다.

## 작업 절차

1. 패키지 관리자, 잠금 파일, 런타임 산출물, 컨테이너 이미지, 생성 코드, 릴리스 게이트를 확인합니다.
2. 직접/전이 의존성 영향, 라이선스 의무, 취약점 악용 가능성, 런타임 노출 범위를 비교합니다.
3. 일반적인 패키지 조언보다 SBOM/출처 증거와 저장소가 정한 감사 명령을 우선합니다.
4. 잠금 파일 일관성, 테스트/빌드, 라이선스 정책, 취약점 스캔 결과, 롤백/갱신 메모를 검증합니다.

## 참고 자료

패키지 추가/갱신, 잠금 파일 변경, CVE 검토, 릴리스 준수 증거 준비 전에는 `references/dependency-supply-chain-checklist.md`를 읽습니다.

릴리스에 SBOM 생성, 라이선스 선택, VEX/예외 처리, 이미지/소스 증명, 산출물 출처 검증이 필요하면 `references/sbom-attestation-release-gate.md`를 읽습니다.
