# 에이전트 도구 위협

애플리케이션, 하네스, 워크플로우, 저장소가 AI 에이전트, MCP 서버, 도구 호출, 프로젝트 로컬 설정, 메모리, 검색, 자동 외부 통신을 포함할 때 이 참고 자료를 사용합니다.

## 핵심 위협 모델

하나의 런타임 안에 아래 세 능력이 함께 있으면 에이전트 위험이 크게 커집니다.

- 개인 또는 내부 데이터: 비밀값, 소스 코드, 고객 데이터, 내부 문서, 로컬 파일, 토큰, 계정 상태, 메모리.
- 신뢰할 수 없는 콘텐츠: 웹 페이지, 이메일, PDF, 스크린샷, OCR 텍스트, 이슈 댓글, PR diff, 의존성 메타데이터, 도구 출력, 모델 생성 텍스트.
- 외부 통신 또는 행동: 네트워크 요청, 채팅 응답, 커밋, 셸 명령, 브라우저 자동화, 티켓 갱신, MCP 도구, 파일 쓰기.

셋이 모두 있으면 프롬프트 주입을 단순한 지시 불복종 문제가 아니라 데이터 유출과 행동 악용 위험으로 취급합니다.

## 점검할 표면

- 프로젝트 로컬 에이전트 설정: hook, 설정, 도구 허용 목록, 환경 오버라이드, MCP 선언, 어댑터 설정.
- 도구 설명과 스키마: 적대적이거나 과도하게 넓은 지시, 숨은 부수 효과, 자격 정보 요청, 읽기/쓰기 동작 불일치.
- 도구 출력: 검색 결과, 로그, 문서, 패키지 메타데이터, OCR, 댓글, 생성 보고서 안에 들어 있는 지시.
- 메모리와 검색: 신뢰할 수 없는 텍스트가 장기 메모리, 출처 레지스트리, 벡터 색인, 캐시, 결정 문서로 승격되는 경로.
- 자동화 권한: 자동 승인, 백그라운드 작업, 예약 에이전트, CI bot, GitHub 댓글, 이슈 할당, PR 검토 흐름.
- 비밀값 경로: 환경 변수, 설정 파일, CLI 인자, 셸 기록, 로그, 브라우저 번들, 스크린샷, 디버그 덤프.

## 가드레일

- 읽기, scaffold, 관리 쓰기, 프로젝트 쓰기 도구를 분리합니다. 쓰기 도구는 명시적 선택, 대상 경로 검증, dry-run 미리보기, apply 플래그, 감사 로그가 있을 때만 노출합니다.
- 저장소를 신뢰하기 전까지 프로젝트 로컬 에이전트 설정을 신뢰하지 않습니다. 신뢰가 확정되기 전에 hook, 셸 설정, MCP 서버를 실행하지 않습니다.
- 신뢰할 수 없는 콘텐츠를 system prompt, 도구 설명, 장기 메모리에 넣지 않습니다. 반드시 써야 한다면 데이터로 표시하고 지시문을 제거합니다.
- 신뢰할 수 없는 콘텐츠와 비밀값을 가진 도구 또는 외부 통신이 결합되기 전에는 명시적인 사람 승인을 요구합니다.
- 비밀값은 최종 출력 직전이 아니라 수집 단계에서 가립니다.
- 메모리 승격은 명시적으로 처리합니다. 런타임 보고서와 검색된 텍스트는 검토 없이 오래 쓸 프로젝트 지식이 되면 안 됩니다.

## 검토 질문

- 신뢰할 수 없는 콘텐츠가 도구 호출, 셸 명령, 네트워크 요청, 커밋, 메시지 전송을 유발할 수 있습니까?
- 저장소가 제어하는 파일이 에이전트 endpoint, 모델 공급자 URL, hook, 도구 허용 목록, 환경 변수를 바꿀 수 있습니까?
- 도구가 작업에 필요한 것보다 더 많은 파일시스템, 네트워크, 자격 정보, 작업 공간 상태를 읽을 수 있습니까?
- 악성 MCP 서버가 도구 설명, 스키마, 리소스 내용, 정상처럼 보이는 결과 안에 지시를 숨길 수 있습니까?
- 생성 보고서나 캐시가 출처와 검토 없이 나중에 메모리로 신뢰될 수 있습니까?
- 고위험 쓰기와 외부 행동에 승인 기록이 남습니까?

## 검증

- 하네스에 파서/어댑터 코드가 있다면 댓글, 문서, 도구 출력, 검색된 콘텐츠 안의 악성 텍스트 fixture 테스트를 추가합니다.
- 기본 MCP/도구 모드가 읽기 전용이고, 선택 플래그가 없으면 쓰기 도구가 숨겨지거나 비활성인지 확인합니다.
- 공개 문서와 생성 예시에 개인 경로, 자격 정보, 공급자 키, 내부 URL, 과도하게 넓은 설정 명령이 없는지 확인합니다.
- 로그가 비밀값을 가리고, 개인 맥락이 섞인 신뢰할 수 없는 원문 프롬프트를 저장하지 않는지 확인합니다.
- CI 또는 bot 흐름에서는 낮은 신뢰 권한 사용자가 사람 승인 없이 특권 에이전트 실행을 유발할 수 없는지 확인합니다.
