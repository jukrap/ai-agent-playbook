# 프로젝트 스킬 정책

프로젝트가 에이전트 스킬 선택 기준을 휴대 가능한 문서로 두고 싶을 때 이 파일을 사용합니다. 작게 유지하고, 커밋하기 전에 프로젝트에 맞게 조정합니다.

## 기본 규칙

- 현재 작업에 명확히 필요한 최소한의 skill만 사용합니다.
- 일반 스킬 조언보다 저장소, 현재 사용자 지시, 실제 코드를 우선합니다.
- 어떤 작업 규칙이 활성화되는지 사용자가 알 수 있도록 선택한 스킬을 짧게 알립니다.
- 설치된 스킬이 있다는 이유만으로 모두 불러오지 않습니다.

## 작업 흐름 스킬

외부 작업 흐름 스킬 묶음은 계획, 디버깅, TDD, 이슈 분류, 아키텍처 검토, 이슈 쪼개기, 인수인계 작업에 유용할 수 있습니다. 이 프로젝트가 명시적으로 요구하지 않는 한 선택적 보조 수단으로 다룹니다.

작업 방식 자체가 주제라면 작업 흐름 스킬을 먼저 사용하고, 그다음 저장소 규칙을 제한하는 가장 작은 프로젝트별 스킬이나 문서를 사용합니다.

## 가져와 쓸 만한 작업 패턴

작업에 맞으면 아래의 작은 패턴을 우선합니다.

- 계획: 저장소에서 확인할 수 있는 사실을 먼저 살핀 뒤, 계획을 실제로 바꿀 수 있는 질문만 합니다.
- 디버깅: 수정안을 내기 전에 재현하거나 빠른 피드백 루프를 좁히고, 가설은 반증 가능하게 세운 뒤 회귀 경로를 검증합니다.
- TDD와 작업 쪼개기: 넓은 수평 변경보다 하나의 동작 또는 하나의 수직 추적 단위씩 진행합니다.
- 프로토타입: 프로토타입은 하나의 불확실성을 확인하는 일회성 산출물로 두고, 끝나면 제거하거나 결과만 실제 구현에 반영합니다.
- 이슈와 PRD 정리: 독립적으로 검증 가능한 수직 단위로 나누고, 수락 기준은 분명하게 쓰며, 오래 stale되기 쉬운 파일 경로 나열은 줄입니다.
- 아키텍처 리뷰: 현재 evidence와 도메인 용어를 기준으로 말하고, 큰 재작성보다 작게 깊어지는 다음 단계를 제안합니다.
- 인수인계: 현재 상태, 검증, 남은 위험, 다음에 실행할 가능성이 큰 명령을 남기고, 민감하거나 local-only인 내용은 공개 문서에 넣지 않습니다.

## 프로젝트 보호 스킬

반복되는 저장소 관심사에는 설치된 playbook 스킬을 사용합니다.

- `repo-onboarding`: 낯설거나 오래된 저장소에 들어갈 때.
- `project-doc-system`: `AGENTS.md`, 명세, 계획, 작업 기록, 로컬 문서를 정리할 때.
- `api-contract-boundary`: API 연동을 변경하거나 검토할 때.
- `style-quality-review`: 재설계 없이 화면 스타일을 검토할 때.
- `commit-worklog-guardrails`: 스테이징, 커밋, 푸시, PR 본문, 작업 기록을 다룰 때.
- `legacy-*`: 숨은 결합이나 호환성 제약이 있는 레거시 시스템을 변경할 때.

## 선택적 구조 검증 도구

저장소 감사 또는 구조 검증 도구는 저장소 전체 구조 주장에 기계적으로 확인한 근거가 필요할 때 도움이 됩니다. 설치되어 있고 기술 스택에 맞을 때만 사용합니다.

- 첫 점검, 오래된 기준선, 큰 리팩터링, 실사 성격의 검토에는 전체 감사를 선호합니다.
- 최신 기준선이 있는 뒤의 작은 국소 질문에는 빠른 후속 확인을 선호합니다.
- 스캔 범위와 최신성을 밝히지 않고 부재, 중복 구조, 죽은 export, 순환 의존성, 정리 대상 개수를 단정하지 않습니다.
- 이 프로젝트가 설치하고 선택한 흐름이 아니라면 작성 전후 자동 게이트가 활성화되어 있다고 가정하지 않습니다.

## Codex 호환성

Claude Code 전용 훅, 슬래시 명령, `CLAUDE_PLUGIN_ROOT` 같은 환경 변수가 Codex에서 동작한다고 가정하지 않습니다. 다른 에이전트 기준으로 작성된 스킬이나 도구라면 의도를 Codex가 지원하는 명령과 저장소 로컬 규칙으로 옮깁니다.

## 스킬 대신 프로젝트 문서를 쓸 때

- 지속적인 저장소 지침은 `AGENTS.md`, 이 파일, 또는 다른 프로젝트 문서에 둡니다.
- 제품 범위는 `PROJECT_SPEC.md`에 둡니다.
- 마일스톤은 `PLANS.md`에 둡니다.
- 긴 로컬 보조 문서는 `.local-ai`, `docs/ai`, `docs/plans` 같은 프로젝트 로컬 폴더에 둡니다.
- 여러 프로젝트에서 재사용할 동작일 때만 스킬을 만들거나 설치합니다.
