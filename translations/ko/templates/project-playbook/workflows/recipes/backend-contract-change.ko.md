# Backend Contract Change

Inputs: API boundary, callers, payload examples, error behavior, compatibility needs.

Outputs: contract note, implementation plan, tests, migration or rollout notes.

Skills: API contract boundary, server-rendered change, auth가 관련되면 security review.

Tools: `contracts check`, `contracts snapshot`, `operator search`, `operator map`.

Stop conditions: 알 수 없는 caller behavior, schema owner 누락, 모호한 error contract.

Verification: contract test, integration test, caller compatibility check.
