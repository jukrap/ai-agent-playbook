# Design Reference Handoff

Inputs: product surface, design intent, brand asset, reference screenshot 또는 board, Figma/mockup source, target screen, implementation boundary, accessibility target.

Outputs: design direction brief, reusable reference finding, brand/token note, image-to-code 또는 Figma handoff contract, visual evidence package, implementation acceptance criteria.

Skills: design brief direction, brand identity system, design reference analysis, image-to-code handoff, design system handoff, visual regression QA, frontend accessibility review.

Tools: reference capability matrix, reference adoption plan, browser 또는 screenshot evidence, `qa_image_diff`, `operator preflight`, `index search`, `write_gate_preview`.

Stop conditions: source authority 불명확, private 또는 unlicensed visual material, product purpose 누락, upstream branding 복사 위험, inaccessible color/motion direction, unsupported responsive target.

Verification: reference boundary 기록, 허용되는 경우 visual evidence package 캡처, token/component mapping 검토, desktop/mobile state matrix 확인, implementation 전 accessibility와 content-fit risk 명명.
