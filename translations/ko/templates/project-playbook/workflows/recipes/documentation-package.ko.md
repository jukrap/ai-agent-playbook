# Documentation Package

Inputs: audience, artifact type, source evidence, changed behavior, user-facing scope, setup commands, owner, freshness date, translation needs, sensitive-data boundary, maintenance path, archive path, known constraints.

Outputs: PRD 또는 lightweight spec, issue plan, release note 또는 changelog, README 또는 docs update, stakeholder package, developer handoff, knowledge package, map update, archive note, open-question list.

Skills: repo onboarding, project doc system, requirements PRD scope review, issue planning triage, release notes changelog, documentation artifact package, ADR spec handoff, durable memory가 영향을 받으면 context engineering memory design.

Tools: `workflow run-preview`, `playbook context`, `operator search`, `operator map`, `canon check`, `index status`, `write-gate preview`, `reference ledger check`, public-doc validation, translation validation, link check, placeholder check.

Stop conditions: unknown audience 또는 artifact type, docs와 code/current memory의 불일치, 알 수 없는 setup 또는 verification path, unreviewed generated evidence, raw transcript promotion, 불명확한 owner 또는 maintenance path, private information risk, translated docs의 translation plan 누락.

Verification: source evidence review, public-doc hygiene, translation coverage, link/path check, placeholder check, setup command check, 현재 file 기준 example validation, maintenance/archive path review, memory가 바뀌면 canon promotion review.
