# Frontend Quality Review

Inputs: affected screen, user flow, supported breakpoint, state/data risk, accessibility target.

Outputs: visual QA note, accessibility finding, state/data risk note, 유용한 경우 screenshot 또는 issue list.

Skills: UI polish, frontend state data flow, frontend accessibility review, visual regression QA.

Tools: browser QA, screenshot diff, `operator preflight`, `index search`, 가능한 경우 project test runner.

Stop conditions: 실행 가능한 화면 누락, 사용할 수 없는 test credential, 불명확한 design baseline, fixture 없는 unstable data source.

Verification: desktop/mobile review, keyboard/focus check, loading/empty/error state, stale data check, overflow와 visual diff check.
