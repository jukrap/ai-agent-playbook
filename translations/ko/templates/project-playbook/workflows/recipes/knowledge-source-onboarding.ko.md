# Knowledge Source Onboarding

Inputs: source type, source owner, use case, privacy tier, credential boundary, update cadence, locator shape, search mode, browse mode, promotion policy.

Outputs: sources json entry, source status note, locator contract, evidence envelope example, browse instruction, freshness caveat, promotion rule, verification command.

Skills: knowledge source registry, knowledge retrieval pipeline review, context engineering memory design, runtime index cache design, data contract lineage review, pre-action fact gate.

Tools: `operator search`, `operator research`, `index status`, `index search`, `reference ledger check`, `canon check`, `diagnostics check`, source browse command when available.

Stop conditions: source owner unknown, credential boundary 불명확, source range unbounded, browse 또는 reopen path unavailable, freshness unknown, locator가 exact evidence를 가리키지 못함, private payload commit 위험, generated evidence가 review 없이 trusted 처리될 위험.

Verification: sources json entry validation, status와 freshness 기록, sample locator reopen, scan range 문서화, credential value 없음, privacy caveat 기록, promotion target review.
